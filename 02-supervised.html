1. 지도학습 개요

지도학습의 정의

지도학습(Supervised Learning)은 정답(레이블)이 미리 정해진 데이터를 이용하여 모델을 학습시키는 머신러닝의 가장 기본적인 방식입니다. 마치 선생님이 "이것이 정답"이라고 알려주며 학생을 가르치는 것처럼, AI 모델은 입력과 정답 쌍을 통해 패턴을 학습합니다.

핵심 용어 - 레이블과 피처
• 레이블(Label): 우리가 예측하려는 정답. 예) "스팸/정상", "고양이/개", "가격(10,000원)"
• 피처(Feature): 예측을 위한 입력 정보. 예) 이메일의 길이, 특정 단어, 발신자 정보
• 데이터셋: (피처, 레이블) 쌍의 모음. 예) (이메일 내용, 스팸 여부)

적응식 신경망과 모수 개념

신경망 모델은 데이터로부터 최적의 파라미터(모수)를 학습하는 적응식 시스템입니다. 학습 과정에서는 두 가지 중요한 개념을 구분해야 합니다.

파라미터 (Parameter) 하이퍼파라미터 (Hyperparameter)
데이터로부터 학습되는 값 학습 전에 사람이 정하는 값
예: 신경망의 가중치(w), 편향(b) 예: 학습률, 은닉층 개수, 배치 크기
모델의 실제 동작 결정 학습 과정 자체를 조절
역전파(Backpropagation)로 업데이트 교차검증이나 경험으로 최적화

6단계 학습절차

모든 지도학습 모델은 다음의 체계적인 절차를 따릅니다:

1
데이터 수집
학습에 필요한 충분한 양의 데이터 준비
2
데이터 전처리
결측값, 이상치 처리, 정규화
3
데이터 분할
학습/검증/테스트 세트로 나누기
4
모델 선택
문제 유형에 맞는 알고리즘 선택
5
모델 학습
손실함수 최소화하여 파라미터 학습
6
평가 및 튜닝
성능 평가 후 하이퍼파라미터 조정
비유: 운전면허 시험
1단계(데이터 수집) = 운전 경험 쌓기
2단계(전처리) = 위험한 상황 구분하기
3단계(분할) = 학습/모의고사/실제시험으로 나누기
4단계(모델 선택) = 어떤 방식으로 배울지 결정 (학원, 친구, 유튜브)
5단계(학습) = 실제 운전 연습
6단계(평가) = 시험 봐서 성적 확인 후 약점 보완
기술사 시험 포인트
• 파라미터와 하이퍼파라미터의 명확한 구분 필수
• 6단계 학습절차를 순서대로 설명할 수 있어야 함
• 적응식(Adaptive) 신경망의 의미: 데이터로부터 자동으로 학습

2. 분류 vs 회귀

지도학습은 예측 대상의 형태에 따라 두 가지로 구분됩니다. 이는 문제의 본질을 이해하는 가장 기본적인 단계입니다.

항목 분류 (Classification) 회귀 (Regression)
정답의 형태 범주형 (Categorical) 연속형 (Continuous)
예측 내용 "무엇인가?" (범주 선택) "얼마나?" (수치 예측)
구체 사례 스팸/정상, 개/고양이/새, 등급(A/B/C) 내일 온도, 주택 가격, 주가
출력값 클래스 또는 확률 (0.1 ~ 1.0) 실수 (음수 가능)
대표 알고리즘 KNN, SVM, 의사결정나무, 로지스틱 회귀 선형회귀, 다항회귀, 신경망
평가 지표 정확도, 정밀도, 재현율, F1점수, AUC MSE, RMSE, MAE, R-squared
이진 vs 다중 이진분류(2개), 다중분류(3개 이상) 일반적으로 1개의 연속값 (다중출력 가능)
비유: 병원 진료
분류 = "이 환자가 독감인가? 감기인가? 폐렴인가?" → 범주 선택
회귀 = "이 환자의 체온은 몇 도인가? 항생제는 몇 mg을 투여해야 하는가?" → 수치 예측
기술사 시험 포인트
• 분류와 회귀의 구분이 명확해야 함
• 이진분류와 다중분류의 구분 필수
• 같은 알고리즘을 분류와 회귀에 모두 사용 가능한 경우도 있음 (의사결정나무, 신경망, SVM)

3. K-최근접 이웃 (K-Nearest Neighbors, KNN)

KNN의 정의
K-최근접 이웃은 새로운 데이터에 대해 "가장 가까운 k개의 이웃 데이터의 클래스를 따른다"는 원칙에 기반한 비모수적 분류 알고리즘입니다.

KNN의 특징

특징 설명
메모리 기반 명시적인 모델을 생성하지 않고 모든 학습 데이터를 메모리에 저장
비모수적 미리 정해진 함수 형태가 없으며, 데이터 형태에 따라 결정경계 결정
Lazy Learning 학습 단계가 없고, 예측 시점에 모든 계산 수행 (느림)
국소적 전역 패턴이 아닌 주변 이웃의 국소 정보만 고려

KNN 절차

거리 계산 → k개 이웃 찾기 → 투표 → 분류

거리 계산 방법

거리 측정 공식 특징
유클리드 거리 √Σ(xi - yi)² 가장 흔함, 직선거리
맨해튼 거리 Σ|xi - yi| 격자 기반, 도시 블록
민코프스키 거리 (Σ|xi - yi|^p)^(1/p) 일반화된 거리
코사인 유사도 cos(θ) = A·B/(|A||B|) 텍스트, 고차원 데이터

K값의 영향

K 값 특징 문제점
K=1 가장 가까운 이웃 1개만 참고 노이즈에 매우 민감 (과적합)
K=3, 5 (작은 값) 결정경계가 복잡하고 유연함 과적합 위험
K=√n (중간값) 일반적으로 권장하는 범위 데이터 크기에 따라 조정 필요
K=n (전체, 큰 값) 결정경계가 단순함 과소적합 (High Bias)
비유: 새 학교에서의 친구 찾기
K=1: 가장 가까이 앉은 학생 1명만 따라함 (그 친구가 나쁜 영향 미칠 수 있음)
K=3: 가장 가까운 3명을 참고 (더 안정적)
K=전체: 반 전체 학생의 성향을 따름 (개성 없음)

KNN의 장단점

장점 단점
구현이 간단하고 직관적 예측이 느림 (모든 학습 데이터와 비교)
비모수적 (데이터 분포 가정 없음) 높은 메모리 사용량
특성공학 최소화 스케일 민감 (정규화 필요)
다중출력, 회귀에도 적용 가능 고차원 데이터에서 거리 의미 부족 (차원의 저주)
기술사 시험 포인트
• 메모리 기반/비모수적/Lazy Learning의 의미 이해 필수
• K값에 따른 편향-분산 트레이드오프
• 거리 측정 방법의 선택이 성능에 미치는 영향
• 차원의 저주: 고차원에서 "가까움"의 의미가 퇴색

4. 서포트 벡터 머신 (Support Vector Machine, SVM)

SVM의 정의
SVM은 두 클래스를 분리하는 최적의 초평면(Hyperplane)을 찾아 최대 마진으로 분류하는 알고리즘입니다. "여유있게 떨어뜨려 분리한다"는 원칙에 기반합니다.

핵심 개념

용어 설명
초평면 (Hyperplane) 두 클래스를 분리하는 경계면. 2D에서는 직선, 3D에서는 평면, nD에서는 초평면
마진 (Margin) 초평면에서 가장 가까운 데이터까지의 거리. 넓을수록 좋음
서포트 벡터 초평면과 가장 가까운 데이터 포인트들. 이들이 경계를 결정함
결정경계 SVM이 분류 결정을 내리는 경계선/면

Hard Margin vs Soft Margin

개념 Hard Margin Soft Margin
정의 모든 데이터를 완벽히 분리 일부 오류를 허용하며 분리
가정 데이터가 선형 분리 가능 데이터가 선형 분리 불가능한 경우도 포함
구현 제약없음 슬랙 변수(Slack Variable) ξ 도입
실무 적용 현실에서는 거의 불가능 일반적으로 사용 (현실성)
일반화 낮음 (과적합 위험) 높음 (더 나은 성능)

커널 트릭 (Kernel Trick)

데이터가 선형 분리 불가능할 때, 원본 특성공간에서 고차원 특성공간으로 암묵적으로 변환하여 분리 가능하게 만드는 기법입니다.

커널 함수 특징
선형 커널 K(x, y) = x·y 선형 분리 가능한 경우 사용
다항식 커널 K(x, y) = (x·y + c)^d 중간 수준의 비선형성
RBF 커널 K(x, y) = exp(-γ||x-y||²) 가장 강력하고 흔히 사용됨
시그모이드 커널 K(x, y) = tanh(κ(x·y) + θ) 신경망과 유사
비유: 원탁과 직선의 분리
2D에서 동심원으로 데이터가 분포하면 직선으로 분리 불가능합니다.
커널 트릭은 이를 3D 공간으로 "펴서" 분리 가능하게 만드는 것입니다.
실제로는 고차원 계산을 하지 않고, 내적(dot product)만 계산하면 됩니다 (계산 효율).
기술사 시험 포인트
• Hard Margin과 Soft Margin의 명확한 구분
• 슬랙 변수 ξ의 역할 이해
• 커널 트릭의 본질: 고차원으로의 함축적 변환
• C 파라미터: 마진 위반 허용도 조절 (C 크면 마진 좁음, C 작으면 마진 넓음)
• RBF 커널이 가장 널리 사용되는 이유

5. 의사결정나무 (Decision Tree)

의사결정나무의 정의
의사결정나무는 데이터를 재귀적으로 분할하여 각 노드에서 선택지를 제시하고, 결국 리프 노드의 클래스로 분류하는 트리 구조의 알고리즘입니다.

나무의 구조

용어 설명
루트 노드 맨 위 시작 노드. 전체 데이터를 첫 번째 분할
내부 노드 분할 조건을 가진 노드. "키 > 170?" 같은 질문
리프 노드 마지막 노드로서 최종 분류 결과를 나타냄
깊이 (Depth) 루트에서 리프까지의 거리. 깊을수록 복잡

분할 기준 (Split Criteria)

각 노드에서 "어느 특성으로 분할할 것인가?"를 결정하기 위한 지표입니다:

지니 불순도 (Gini Impurity)
Gini = 1 - Σ(pi)² where pi는 i번째 클래스의 비율
• 0: 완전히 순수 (한 클래스만 존재)
• 0.5: 완전히 섞임 (이진분류에서 50:50)
• 계산 빠르고 CART에서 사용
정보이득 (Information Gain)
IG = Entropy(Parent) - Σ(|Child|/|Parent| × Entropy(Child))
• Entropy = -Σ(pi × log2(pi)): 무질서한 정도
• 정보이득이 큰 분할을 선택 (엔트로피 감소 최대)
• ID3, C4.5에서 사용
카이제곱 검정 (Chi-Square)
χ² = Σ((O - E)² / E) where O는 관측, E는 기대
• 예상과 실제의 차이 측정
• CHAID에서 사용

나무 성장 알고리즘

알고리즘 분할 기준 특징
ID3 정보이득 범주형만 처리, 이진분할 강제
C4.5 정보이득비 ID3 개선, 연속형 처리 가능
CART 지니지수 이진분할만, 회귀나무도 가능 (현대적)
CHAID 카이제곱 다진분할 가능

가지치기 (Pruning)

나무가 너무 깊어지면 학습 데이터에만 과도하게 적합되는 과적합이 발생합니다. 가지치기는 나무를 잘라서 일반화 성능을 높입니다.

가지치기 방식 설명 시점
선제적 가지치기 (Pre-pruning) 더 이상 분할하지 않기 나무 성장 중
사후 가지치기 (Post-pruning) 먼저 전체 나무를 키운 후 자르기 나무 성장 후
비유: 병원 진단
의사가 진료할 때: 1. "열이 있는가?" → 예/아니오 (루트 노드)
2. "38도 이상인가?" → 예/아니오 (내부 노드)
3. "최근 여행 이력이 있는가?" → 예/아니오
4. "독감" 또는 "감기" (리프 노드)

이것이 의사결정나무입니다. 너무 세밀하면 과진단 위험 (가지치기 필요).
기술사 시험 포인트
• 지니지수 vs 정보이득의 차이 (계산 복잡도, 사용 알고리즘)
• CART가 가장 현대적이고 회귀나무도 가능
• 엔트로피의 개념: 정보이론에 기반
• 과적합 방지 방법: 깊이 제한, 최소 샘플 수, 가지치기
• 나무의 해석가능성이 강점

6. 로지스틱 회귀 (Logistic Regression)

이름과의 혼동
이름에 "회귀"가 있지만, 로지스틱 회귀는 분류 알고리즘입니다.
선형회귀의 출력을 S자형 함수(시그모이드)에 통과시켜 확률로 변환합니다.

핵심 변환 과정

1단계: 오즈 (Odds)
Odds = P(Event) / P(Not Event) = p / (1-p)
• 사건이 일어날 확률을 일어나지 않을 확률로 나눈 값
• 예) 로또 1등 확률이 1/1000이면, Odds = 1/999 ≈ 0.001
2단계: 로짓 (Logit)
Logit = ln(Odds) = ln(p / (1-p))
• 오즈에 자연로그를 취한 값
• 범위: -∞ ~ +∞ (선형회귀 가능)
• 선형회귀로 예측하기 위해 변환
3단계: 시그모이드 (Sigmoid)
p = 1 / (1 + e^(-z)) where z = w₀ + w₁x₁ + ... + wₙxₙ
• 로짓을 역으로 변환하여 확률(0~1)로 변환
• S자형 곡선
• 모든 입력을 0~1 사이의 확률로 변환

로지스틱 회귀의 과정

선형식 → 로짓 변환 → 시그모이드 → 확률 → 분류

판별함수

개념 설명
선형 판별함수 z = w₀ + w₁x₁ + w₂x₂ + ... + wₙxₙ
이것을 시그모이드에 통과: p = σ(z)
의사결정 경계 z = 0 또는 p = 0.5인 지점
이 경계선이 두 클래스를 분리
선형성 로지스틱 회귀의 결정경계는 항상 선형
비선형 패턴은 특성공학 필요

손실함수: 교차엔트로피

개념 설명
이진분류 L = -[y×log(ŷ) + (1-y)×log(1-ŷ)]
y는 실제값(0 또는 1), ŷ는 예측확률
최적화 경사하강법(Gradient Descent)으로 가중치 w 업데이트
폐쇄형 해가 없음 (신경망과 유사)
규제 L1(Lasso) 또는 L2(Ridge) 규제로 과적합 방지
가중치 크기 제약
비유: 합격 확률 예측
대학 입시에서 수능점수, GPA, 자소서 질을 고려해 합격 확률을 예측합니다.
점수가 높을수록 확률이 높아지지만, 정확히는 S자 곡선을 따릅니다.
100점이든 200점이든 확률은 0~1 사이에서만 변함 (시그모이드).
기술사 시험 포인트
• 오즈(Odds), 로짓(Logit), 시그모이드의 정확한 의미와 변환 과정
• 로지스틱 회귀가 분류이지 회귀가 아닌 이유
• 결정경계가 항상 선형이라는 한계
• 교차엔트로피 손실함수
• 다중분류 확장: Softmax 함수 사용

7. 나이브 베이즈 (Naive Bayes)

나이브 베이즈의 정의
나이브 베이즈는 베이즈 정리를 기반으로 각 특성의 조건부 독립을 가정하는 생성 모델입니다.
"나이브(naive)"는 특성들이 서로 독립이라는 단순한 가정을 의미합니다.

베이즈 정리 (Bayes' Theorem)

공식
P(C|X) = P(X|C) × P(C) / P(X)

where:
• P(C|X): 후사확률 (Posterior) - X가 주어졌을 때 클래스 C의 확률
• P(X|C): 우도 (Likelihood) - 클래스 C가 주어졌을 때 X의 확률
• P(C): 사전확률 (Prior) - 클래스 C의 사전 확률
• P(X): 증거 (Evidence) - X의 확률 (정규화 상수)

생성 모델 (Generative) vs 판별 모델 (Discriminative)

특징 생성 모델 판별 모델
학습 대상 P(X|C)를 학습
(각 클래스의 데이터 분포)		 P(C|X)를 직접 학습
(결정경계)
방식 각 클래스의 분포를 모델링 후 베이즈 정리 적용 데이터로부터 직접 경계 학습
장점 데이터 적을 때도 가능
새로운 클래스 추가 용이		 보통 더 정확함
계산 효율적
단점 분포 가정이 틀리면 성능 저하 충분한 학습 데이터 필요
예시 나이브 베이즈, HMM 로지스틱 회귀, SVM, 신경망

나이브 가정 (Conditional Independence)

나이브 가정
모든 특성이 클래스 주어졌을 때 조건부 독립이라고 가정합니다.

P(X₁, X₂, ..., Xₙ|C) = P(X₁|C) × P(X₂|C) × ... × P(Xₙ|C)

이는 현실에서 대부분 거짓이지만, 단순화로 계산 가능하고 실제로 잘 작동합니다.

나이브 베이즈의 종류

종류 조건부 확률 분포 사용 사례
가우시안 정규분포 가정
P(X|C) ~ N(μ, σ²)		 연속형 변수 (신장, 온도)
다항분포 이산 카운트 데이터
P(X|C) ~ Multinomial		 텍스트 분류 (단어 빈도)
베르누이 이진 특성
P(X|C) ~ Bernoulli		 이진 특성 (있음/없음)
비유: 스팸 메일 필터
"Nigerian Prince"라는 단어가 있으면 스팸일 확률이 높습니다.
"무료"라는 단어도 있으면 스팸 확률이 더 높아집니다.
나이브 베이즈는 각 단어의 영향을 독립적으로 계산하여 최종 확률을 구합니다
(실제로는 단어들이 연관되어 있지만, 단순화로 계산 효율 높임).
기술사 시험 포인트
• 베이즈 정리의 각 항의 의미
• 생성 모델 vs 판별 모델의 명확한 구분
• 나이브 가정이 현실에서 거짓이지만 왜 작동하는가
• 3가지 나이브 베이즈 종류 (가우시안, 다항분포, 베르누이)
• 데이터 부족할 때 생성 모델이 유리한 이유

8. 앙상블 학습 (Ensemble Learning)

앙상블의 정의
여러 개의 약한 학습기(Weak Learner)를 조합하여 강한 학습기(Strong Learner)를 만드는 방법입니다.
"여럿이 하나보다 낫다"는 민주주의 원칙을 머신러닝에 적용합니다.

주요 앙상블 방법 비교

방법 방식 학습 개선 대상 예시
배깅 병렬 생성 복원추출로 여러 데이터셋 생성 분산(Variance) Random Forest
부스팅 순차 생성 이전 모델의 오류를 다음 모델이 보정 편향(Bias) AdaBoost, XGBoost
스태킹 계층적 메타 모델이 기반 모델들의 예측을 입력으로 편향과 분산 Stacking Classifier
블렌딩 계층적 (단순화) 스태킹과 유사하지만 더 간단함 편향과 분산 Blending
보팅 병렬 결합 여러 모델의 예측을 다수결 분산 Voting Classifier

배깅 (Bootstrap Aggregating)

비유: 여론조사
국민 전체를 조사할 수 없으므로,
같은 크기의 샘플을 여러 번 뽑아(복원추출)
각 샘플에서 여론을 물은 후
평균을 내는 것과 같습니다.
배깅의 원리
1. 원본 데이터에서 복원추출로 n개의 부트스트랩 샘플 생성
2. 각 샘플마다 동일한 모델 학습
3. 분류: 다수결 투표, 회귀: 평균
4. 분산 감소로 일반화 성능 개선

부스팅 (Boosting)

부스팅의 원리
1. 첫 번째 약한 학습기 학습
2. 그것이 못 맞춘 데이터에 높은 가중치 부여
3. 두 번째 학습기는 어려운 데이터에 집중
4. 이를 반복하여 순차적으로 강력한 모델 구성
비유: 보충학습
1차 시험에서 못한 부분을 선생님이 파악한 후,
2차 학습에서 그 부분만 집중 가르칩니다.
이를 반복하면 점점 더 강한 학생이 됩니다.

스태킹 (Stacking) vs 블렌딩 (Blending)

특징 스태킹 블렌딩
메타 모델 학습 교차검증으로 생성된 메타 특성 홀드아웃 검증 세트의 예측
계산 비용 높음 (K-폴드 반복) 낮음 (한 번만)
과적합 위험 낮음 (교차검증 기반) 약간 높음
실무 사용 정확도 중시 속도 중시
비유: 법원 항소
1심 판사(기반 모델1), 2심 판사(기반 모델2)가 각각 판결하면,
대법원 판사(메타 모델)가 그 판결들을 검토해 최종 결정을 내립니다.

보팅 (Voting)

보팅 방식 설명 사용
하드 보팅 각 모델의 예측 클래스를 모아 다수결 분류 (Yes/No)
소프트 보팅 각 모델의 예측 확률을 평균 분류 (확률 평균)
회귀 여러 모델의 예측값을 평균 회귀
기술사 시험 포인트
• 배깅의 분산 감소 원리
• 부스팅의 순차적 학습과 편향 감소
• 스태킹과 블렌딩의 차이 (메타 특성 생성 방식)
• 하드 보팅 vs 소프트 보팅
• 언제 어느 방법을 사용할지 판단 능력

9. 랜덤 포레스트 (Random Forest)

랜덤 포레스트의 정의
배깅과 의사결정나무, 그리고 특성의 무작위 선택을 결합한 강력한 앙상블 알고리즘입니다.
"숲"은 수많은 "나무"(의사결정나무)들의 집합을 의미합니다.

핵심 개념

개념 설명
부트스트랩 샘플링 N개 데이터에서 복원추출로 N개를 여러 번 샘플링
약 63% 데이터만 각 나무에 사용 (OOB: Out-of-Bag)
특성 랜덤 선택 각 노드에서 모든 특성 대신 √d개(분류) 또는 d/3개(회귀) 랜덤 선택
나무 다양성 증가
최대 깊이 제한 없음 각 나무를 완전히 성장시킴 (가지치기 없음)
낮은 편향
병렬 처리 각 나무는 독립적으로 생성되어 병렬화 가능

랜덤 포레스트의 절차

부트스트랩 → 나무 성장 → 특성 무작위 → 예측 결합

특성 중요도 (Feature Importance)

계산 방법
1. 각 특성이 전체 포레스트의 모든 분할에 기여한 정도 측정
2. 지니지수 또는 정보이득 기반 계산
3. 특성별 누적 기여도를 정규화하여 중요도 산출
4. 어떤 특성이 분류에 가장 영향을 미치는지 파악 가능

OOB (Out-of-Bag) 추정

OOB 추정의 장점
• 별도의 검증 세트 불필요
• 각 샘플마다 약 37%는 사용되지 않음 (OOB)
• 이 OOB 샘플로 검증하여 일반화 성능 추정 가능
• 테스트 성능과 유사한 정확도 제공

랜덤 포레스트의 장단점

장점 단점
높은 정확도 (강력한 성능) 해석 어려움 (나무 개수가 많음)
특성 중요도 제공 계산 비용 높음
과적합 저항성 우수 메모리 사용량 많음
병렬 처리 가능 임의성으로 결과 재현성 낮음
특성 정규화 불필요 불균형 데이터에 약할 수 있음
비유: 영화 평점
100명의 영화 평론가가 각각 독립적으로 영화를 평가하고 (부트스트랩),
각 평론가는 약간 다른 기준으로 평가합니다 (특성 무작위 선택),
최종 평점은 100명의 평가의 평균입니다 (앙상블)
개별 평론가의 의견보다 훨씬 신뢰할 수 있습니다.
기술사 시험 포인트
• 부트스트랩 샘플링의 약 63% 선택 비율
• 특성 무작위 선택의 개수: √d (분류) vs d/3 (회귀)
• OOB 추정의 개념과 장점
• 특성 중요도 계산 방식
• 랜덤 포레스트와 배깅 의사결정나무의 차이

10. 부스팅 알고리즘 상세

부스팅은 순차적으로 약한 학습기를 결합하여 강한 학습기를 만드는 기법입니다. 여러 종류가 있으며, 각각의 특성을 이해하는 것이 중요합니다.

주요 부스팅 알고리즘 비교

알고리즘 가중치 업데이트 손실함수 특징
AdaBoost 오분류 샘플에 지수 가중치 지수 손실 (Exponential) 부스팅의 기초, 구현 간단
Gradient Boosting 손실함수 그래디언트 방향 임의 (제곱, 로그 등) 더 강력, 다양한 손실함수 지원
XGBoost 테일러 전개 2차 도함수 정규화 포함 목적함수 병렬화, 정규화, 빠른 수렴
LightGBM 리프 방향 성장, 정규화 그래디언트 기반 더 빠름, 메모리 효율, 범주형 지원
CatBoost 범주형 특성 자동 처리 범주형 기반 손실 범주형 데이터 최적화, 과적합 저항

AdaBoost (Adaptive Boosting)

AdaBoost의 원리
1. 모든 데이터에 동일 가중치로 시작
2. 첫 약한 학습기 학습, 오분류 데이터 가중치 증가
3. 다음 학습기는 어려운 데이터에 집중
4. 각 학습기의 가중치: α = 0.5 × ln((1-ε)/ε) (ε는 오분류율)
5. 최종 예측: 가중치된 투표

Gradient Boosting (GB)

Gradient Boosting의 핵심
• 각 단계에서 손실함수의 음의 그래디언트 방향으로 새 모델 학습
• 이전 모델의 잔차(Residual)를 피팅
• 임의의 미분 가능한 손실함수 사용 가능
• 수렴이 느리지만 높은 정확도

XGBoost (eXtreme Gradient Boosting)

XGBoost의 개선사항
• 테일러 전개로 2차 도함수 활용 (더 정확한 업데이트)
• 정규화 포함: L1(Lasso) + L2(Ridge) 페널티
• 병렬 처리 가능
• 결측값 자동 처리
• Early Stopping으로 최적 반복 횟수 자동 결정

LightGBM (Light Gradient Boosting Machine)

특징 설명
Leaf-wise 성장 트리 깊이 우선이 아닌 리프 중심 성장 (더 빠른 수렴)
범주형 특성 범주형 특성을 원-핫 인코딩 없이 직접 처리
메모리 효율 히스토그램 기반 학습으로 메모리 50% 이하
속도 XGBoost보다 10~20배 빠름

CatBoost (Categorical Boosting)

CatBoost의 특징
• 범주형 변수를 자동으로 인코딩
• 일반화 오류 감소: 순서화된 부스팅
• 과적합 저항성 우수
• 하이퍼파라미터 튜닝 덜 필요
• GPU 지원
기술사 시험 포인트
• AdaBoost부터 CatBoost까지의 진화 과정
• 그래디언트 부스팅의 기본 원리
• XGBoost: 2차 도함수, 정규화, 병렬화
• LightGBM: 속도 개선 (히스토그램, Leaf-wise)
• CatBoost: 범주형 특성 최적화
• 실무에서 언제 어느 것을 사용할지 판단

11. XGBoost 상세

XGBoost의 목적함수
L(θ) = Σ l(yi, ŷi) + Σ Ω(ft)

where:
• l(yi, ŷi): 손실함수 (오류 측정)
• Ω(ft): 정규화 항 (모델 복잡도 제어)

정규화 (Regularization)

정규화 항 설명 효과
L1 정규화 Σ |w| (가중치 절대값) 특성 선택: 중요하지 않은 가중치를 0으로
L2 정규화 Σ w² (가중치 제곱) 가중치 축소: 모든 가중치를 작게
트리 정규화 γ × T + λ × Σ w² (T는 리프 개수) 나무 복잡도 제어

XGBoost의 핵심 파라미터

파라미터 범위 설명
eta (학습률) 0 ~ 1 (기본 0.3) 각 부스팅 단계에서의 수축. 작을수록 안정적이지만 느림
max_depth 1 ~ ∞ (기본 6) 트리 최대 깊이. 과적합 방지
min_child_weight 0 ~ ∞ (기본 1) 리프의 최소 가중치. 크면 보수적 (과적합 방지)
subsample 0 ~ 1 (기본 1) 각 나무마다 샘플 비율 (0.5~1 추천)
colsample_bytree 0 ~ 1 (기본 1) 각 나무마다 특성 샘플 비율
lambda (L2 정규화) 0 ~ ∞ (기본 1) L2 페널티. 크면 가중치 축소
alpha (L1 정규화) 0 ~ ∞ (기본 0) L1 페널티. 특성 선택 효과

병렬연산 (Parallelization)

XGBoost의 병렬화
• 트리 구축 시 각 노드에서의 최적 분할을 병렬로 탐색
• 각 특성의 분할점을 병렬로 계산
• GPU 지원으로 더욱 빠른 처리
• 반복 간에는 순차 처리 (부스팅의 특성)

Early Stopping

Early Stopping의 메커니즘
1. 검증 성능을 모니터링하며 모델 학습
2. 검증 성능이 연속 n번(patience) 개선되지 않으면 중단
3. 최적의 반복 횟수에서 멈춤
4. 과적합 방지와 계산 시간 단축

예: 100회 반복 설정, early_stopping_rounds=10이면
60회 차에서 최고 성능이 나면, 70회 차에서 멈춤

XGBoost의 학습 과정

초기화 → 그래디언트 계산 → 나무 구축 → 가중치 업데이트 → Early Stopping 확인
기술사 시험 포인트
• XGBoost의 목적함수: 손실 + 정규화
• 테일러 전개를 통한 2차 도함수 활용
• 주요 파라미터의 역할 (eta, max_depth, subsample 등)
• Early Stopping의 기준과 효과
• 병렬화 가능한 부분과 불가능한 부분 (순차 부스팅)

12. 교차검증 (Cross-Validation)

데이터 분할이 작으면 모델의 성능 평가가 불안정합니다. 교차검증은 제한된 데이터를 효율적으로 활용하여 신뢰할 수 있는 성능 평가를 제공합니다.

문제: 데이터 부족과 과적합

비유: 학생 평가
1. 교사가 시험 문제만으로 학생 실력 평가 → 부정확
2. 다양한 시험으로 여러 번 평가 → 더 정확
3. 최종 평가: 여러 시험의 평균

교차검증도 같은 원리입니다.

주요 교차검증 기법

기법 절차 특징
Hold-out 학습(60~70%) / 검증(10~15%) / 테스트(15~20%) 가장 간단, 데이터 낭비, 불안정
K-Fold k등분하여 k번 반복 (1개 검증, k-1개 학습) 표준, 데이터 활용 효율적, k=5 또는 10
Stratified K-Fold K-Fold + 클래스 비율 유지 불균형 데이터에 필수
Leave-One-Out (LOO) k=n (데이터 개수) 편향 낮음, 계산 비용 높음 (n번 반복)
Leave-P-Out (LPO) p개를 제외하고 학습 LOO보다는 효율적, 여전히 비쌈
Time Series Split 시간 순서 유지한 분할 시계열 데이터 전용

K-Fold의 절차

데이터 5등분 → Fold1 검증 → Fold2 검증 → ... → 평균 성능

Stratified K-Fold

언제 필요한가?
• 불균형 데이터: 암진단 데이터가 "암 1%, 정상 99%"
• 일반 K-Fold: 어떤 폴드는 암 0개, 다른 폴드는 암 10개 (불균형)
• Stratified K-Fold: 각 폴드에서 1% / 99% 비율 유지
• 결과: 안정적이고 신뢰할 수 있는 성능 평가

Leave-One-Out (LOO)

특징 설명
k 값 k = n (데이터 개수)
반복 횟수 n번 (매우 많음)
장점 편향 가장 낮음, 데이터 최대 활용
단점 계산 비용 극도로 높음, 노이즈 민감
사용 소규모 데이터셋에만 (n < 1000)
기술사 시험 포인트
• Hold-out vs K-Fold의 장단점
• Stratified K-Fold가 필수인 상황
• LOO의 계산 비용과 편향
• 일반적인 권장사항: K=5 또는 K=10
• 시계열 데이터는 시간 순서 유지 필수
• 하이퍼파라미터 튜닝 시 중첩 교차검증 사용

13. 분류 모델 평가

분류 문제에서 정확도(Accuracy)만으로는 부족합니다. 다양한 관점에서 모델 성능을 평가해야 합니다.

혼동행렬 (Confusion Matrix)

예측
양성 (1) 음성 (0)
실제 양성 (1) TP (참양성) FN (거짓음성)
음성 (0) FP (거짓양성) TN (참음성)
각 항의 의미
• TP (True Positive): 양성이라고 예측했는데 실제로 양성 ✓
• TN (True Negative): 음성이라고 예측했는데 실제로 음성 ✓
• FP (False Positive): 양성이라고 예측했는데 실제로 음성 ✗ (거짓알람)
• FN (False Negative): 음성이라고 예측했는데 실제로 양성 ✗ (놓친 것)

성능 지표들

지표 공식 의미 사용 시기
정확도 (Accuracy) (TP + TN) / (TP + FP + FN + TN) 전체 중 맞춘 비율 균형 데이터
정밀도 (Precision) TP / (TP + FP) "양성"이라 한 것 중 진짜 비율 거짓양성 비용 높을 때
재현율 (Recall) TP / (TP + FN) 실제 양성 중 찾은 비율 거짓음성 비용 높을 때
F1점수 2 × (P × R) / (P + R) 정밀도와 재현율의 조화평균 둘의 균형 필요할 때
F-Beta (1+β²) × (P × R) / (β² × P + R) 재현율을 더 중요하게 (β>1) 비대칭 가중치
특이도 (Specificity) TN / (TN + FP) 실제 음성 중 찾은 비율 의료진단

정밀도-재현율 트레이드오프

상황 정밀도 중시 재현율 중시
스팸 필터 정상 메일을 스팸으로 보내면 안 됨 (재현율은 낮아도 괜찮음)
암진단 (정밀도는 낮아도 괜찮음) 암을 놓치면 안 됨 (정밀도 낮음 용인)
신용카드 사기 정상을 사기로 꼬리표하면 고객 불만 사기를 놓치면 손실 발생

ROC 곡선과 AUC

ROC (Receiver Operating Characteristic)
• X축: FPR (거짓양성율) = FP / (FP + TN)
• Y축: TPR (참양성율) = TP / (TP + FN) = 재현율
• 임계값을 변경하면서 그린 곡선
• AUC (곡선 아래 면적): 0.5 (운) ~ 1.0 (완벽)
PR (Precision-Recall) 곡선
• X축: 재현율, Y축: 정밀도
• 불균형 데이터에서 ROC보다 더 유용
• 마찬가지로 AUC로 평가
• ROC보다 작은 규모로 변하여 차이 민감

임계값 조정

임계값의 영향
로지스틱 회귀나 확률 기반 분류는 기본 임계값 0.5를 사용합니다.

임계값을 낮추면 (예: 0.3):
• 양성 판정이 더 쉬워짐 → 재현율↑, 정밀도↓

임계값을 높이면 (예: 0.7):
• 양성 판정이 더 어려워짐 → 재현율↓, 정밀도↑

향상도 차트 (Lift Chart)

향상도 차트의 사용
마케팅 분야에서 모델의 효율성을 평가합니다.

Lift = 모델의 성과 / 무작위 성과

예) 무작위로 고객을 선택하면 10% 반응
모델로 상위 30% 고객 선택 → 25% 반응
Lift = 25% / 10% = 2.5배

코언의 카파 지수 (Cohen's Kappa)

카파 지수
κ = (Po - Pe) / (1 - Pe)

• Po: 관찰된 일치도
• Pe: 우연에 의한 기대 일치도
• -1 ~ 1 범위, 0.4 이상이 보통 수준
• 불균형 데이터에서 정확도보다 신뢰할 수 있음
기술사 시험 포인트
• 혼동행렬의 TP, TN, FP, FN 정의
• 정밀도와 재현율의 트레이드오프 (임계값 조정)
• 균형 데이터: ROC/AUC 사용
• 불균형 데이터: PR 곡선 또는 카파 지수
• 비즈니스 목표에 따른 지표 선택
• 향상도 차트의 마케팅 활용

14. 회귀 모델 평가

회귀 문제의 평가는 예측 오류의 크기를 측정하는 지표들을 사용합니다. 절대오차와 제곱오차의 두 가지 관점이 있습니다.

절대오차 지표

지표 공식 특징
AE (Absolute Error) |y - ŷ| 개별 오류, 최대값이나 평균 계산
MAE (Mean Absolute Error) (1/n) Σ|yi - ŷi| 평균 절대오차, 원래 단위 유지, 해석 쉬움
MAPE (Mean Absolute Percentage Error) (1/n) Σ|yi - ŷi| / |yi| × 100 백분율 오류, 크기 무관한 비교
RMAPE 중앙값 기반 MAPE 이상치 영향 적음

제곱오차 지표

지표 공식 특징
MSE (Mean Squared Error) (1/n) Σ(yi - ŷi)² 큰 오류에 더 높은 패널티, 미분 가능
RMSE (Root Mean Squared Error) √MSE 원래 단위로 변환, 가장 널리 사용
MSLE (Mean Squared Log Error) (1/n) Σ(log(yi+1) - log(ŷi+1))² 상대적 오류 중시, 음수 값 처리 불가

상대적 오류 지표

지표 공식 사용
R² (결정계수) 1 - (SS_res / SS_tot)
where SS_res = Σ(yi - ŷi)²
SS_tot = Σ(yi - ȳ)²		 설명력, 0 ~ 1 범위 (1이 최고)
음수 가능 (모델이 평균보다 나쁨)
Adjusted R² 1 - (1 - R²) × (n-1) / (n-p-1) 특성 개수에 따른 보정
특성 추가 효과 평가
RMSLE (Root Mean Squared Log Error) √[(1/n) Σ(log(yi+1) - log(ŷi+1))²] 상대적 오류, 스케일 불변

MAE vs RMSE 선택

특징 MAE RMSE
계산 절대값 제곱근
이상치 민감도 낮음 (선형) 높음 (제곱)
최적화 중앙값 평균값
사용 이상치 있는 데이터 정규분포 데이터
해석 더 직관적 통계학에서 선호

R² (결정계수) 이해

R² 의미
R² = 1 - (모델의 오차) / (단순 평균의 오차)

• R² = 1.0: 완벽한 적합
• R² = 0.8: 80% 설명력 (좋음)
• R² = 0.5: 50% 설명력 (중간)
• R² = 0.0: 모델이 평균만큼만 설명
• R² < 0: 모델이 평균보다 나쁨

다중회귀에서의 조정 R²

Adjusted R²의 필요성
특성을 추가하면 항상 R²가 증가합니다 (과적합).

Adj R² = 1 - (1 - R²) × (n-1) / (n-p-1)

where n = 샘플 수, p = 특성 개수

불필요한 특성 추가 시 감소 → 모델 선택에 유용

스케일에 따른 지표 선택

상황 추천 지표 이유
절대 오류값이 중요 MAE 해석이 직관적 (원 단위)
상대적 오류 비교 MAPE, R² 스케일 무관, 여러 모델 비교
이상치가 많은 데이터 MAE, MSLE 큰 오류의 영향 제한
모델 설명력 강조 R² (또는 Adj R²) 데이터의 변동성 설명도
학습 과정 최적화 MSE (또는 RMSE) 미분 가능하여 경사하강법 사용
비유: 화살 맞추기
• MAE: 평균적으로 과녁에서 얼마나 떨어져 있는가?
• RMSE: 운이 안 좋으면 얼마나 멀리 빗나가는가? (큰 오류 강조)
• R²: 전체 움직임의 몇 %를 설명하는가? (설명력)
기술사 시험 포인트
• MAE vs RMSE의 차이와 선택 기준
• R² 의 의미와 해석 (0 ~ 1 또는 음수 가능)
• Adjusted R² 의 필요성 (특성 수 보정)
• MAPE의 사용: 상대적 오류, 스케일 무관
• 이상치가 있을 때 MAE 선호
• 통계 기반 분석은 RMSE 선호

이 장의 핵심 요점

  1. 지도학습은 정답이 있는 데이터로부터 학습하는 기본 방식
  2. 파라미터는 학습되는 값, 하이퍼파라미터는 미리 정하는 값
  3. 분류와 회귀는 예측 대상의 형태에 따른 구분
  4. KNN: 가까운 이웃의 투표, 비모수적, 메모리 기반
  5. SVM: 최대 마진으로 분리, 커널 트릭으로 비선형 처리
  6. 의사결정나무: 재귀적 분할, 해석 가능, 가지치기로 과적합 방지
  7. 로지스틱 회귀: 선형식을 시그모이드로 확률화, 판별 모델
  8. 나이브 베이즈: 베이즈 정리 기반, 생성 모델, 조건부 독립 가정
  9. 배깅: 병렬 앙상블, 분산 감소 (예: 랜덤 포레스트)
  10. 부스팅: 순차 앙상블, 편향 감소 (예: AdaBoost, XGBoost)
  11. XGBoost: 2차 도함수, 정규화, Early Stopping, 병렬화
  12. 교차검증: K-Fold (k=5 또는 10), Stratified K-Fold (불균형 데이터)
  13. 분류 평가: 정밀도/재현율 트레이드오프, ROC/AUC, 카파 지수
  14. 회귀 평가: MAE (직관적), RMSE (통계), R² (설명력)
  15. 불균형 데이터는 PR 곡선과 F1 점수 사용
