1. GAN: 생성적 적대 신경망

GAN의 기본 개념

GAN (Generative Adversarial Network, 생성적 적대 신경망)은 생성자(Generator)와 판별자(Discriminator)라는 두 개의 신경망이 서로 경쟁하면서 학습하는 모델입니다. 생성자는 노이즈로부터 가짜 데이터를 만들려고 노력하고, 판별자는 진짜와 가짜를 구별하려고 노력합니다.

GAN은 2014년 Ian Goodfellow가 제안한 혁신적인 기술입니다. 기존의 신경망은 주로 분류나 예측 같은 판별 작업에 사용되었지만, GAN은 처음으로 신경망으로 새로운 데이터를 생성할 수 있게 만들었습니다.

비유로 이해하기: GAN은 위조지폐 제조범과 경찰의 대결 같습니다. 위조범(생성자)은 더 완벽한 가짜 지폐를 만들려고 노력하고, 경찰(판별자)은 진짜와 가짜를 구분하려고 노력합니다. 이런 경쟁 속에서 위조범은 점점 더 정교한 지폐를 만들게 되는 것입니다.

생성자(Generator)와 판별자(Discriminator)

생성자의 역할

생성자는 무작위 노이즈(잡음) 벡터 z를 입력으로 받아서 진짜처럼 보이는 가짜 데이터 G(z)를 생성합니다. 예를 들어, 이미지 생성 GAN이라면, 생성자는 랜덤 벡터를 받아서 실제 사진처럼 보이는 이미지를 만드는 것입니다.

판별자의 역할

판별자는 이미지를 입력으로 받아서 "이것이 진짜일 확률은 얼마인가?"라는 질문에 답합니다. D(x)는 0부터 1 사이의 확률값을 출력하며, 1에 가까울수록 "진짜"라고 생각하는 것입니다.

적대적 학습(Adversarial Learning)

노이즈 z 생성 → 생성자 G(z) → 가짜 데이터 → 판별자 D → 피드백
1
생성
생성자가 노이즈로부터 가짜 데이터를 만듭니다.
2
판별
판별자가 진짜와 가짜를 구분하려고 합니다.
3
손실 계산
두 신경망의 오류를 계산합니다.
4
역전파
가중치를 업데이트합니다.

Min-Max 게임

GAN의 학습 목표는 다음과 같이 표현됩니다:

  • 생성자의 목표: 판별자가 "이건 진짜야!"라고 생각하도록 속이기 (손실 최소화)
  • 판별자의 목표: "이건 가짜야!"라고 정확하게 구분하기 (손실 최소화)

이를 수식으로 표현하면, 생성자는 판별자의 실수를 최대화하려고 하고, 판별자는 자신의 실수를 최소화하려고 합니다. 이는 "최소-최대 게임(minimax game)"으로 불리며, 균형에 도달할 때 생성자는 진정한 데이터 분포를 학습하게 됩니다.

GAN의 주요 변형(Variants)

DCGAN (Deep Convolutional GAN): 합성곱 신경망(CNN)을 생성자와 판별자에 사용한 GAN입니다. 이미지 생성에 매우 효과적이며, 고해상도 이미지 생성의 기초가 되었습니다.
StyleGAN: 이미지의 스타일을 세밀하게 제어할 수 있는 GAN입니다. 스타일과 내용을 분리하여, 얼굴의 스타일은 유지하면서 특정 특징만 수정할 수 있습니다. 얼굴 생성 분야에서 매우 유명합니다.
CycleGAN: 한 도메인의 이미지를 다른 도메인으로 변환합니다. 쌍을 이루는 이미지 없이도 학습 가능합니다. 예: 말 사진 ↔ 얼룩말 사진 변환, 낮 사진 ↔ 밤 사진 변환.
Pix2Pix: 입력 이미지를 출력 이미지로 직접 변환합니다. 쌍을 이루는 이미지 데이터가 필요합니다. 예: 선화 → 컬러 사진, 스케치 → 실제 사진.
WGAN (Wasserstein GAN): Wasserstein 거리를 사용하여 학습의 안정성을 크게 개선했습니다. 기존 GAN의 모드 붕괴 문제를 효과적으로 해결합니다.
StarGAN: 여러 속성을 동시에 변환할 수 있는 GAN입니다. 하나의 신경망으로 "젊은 남성 → 나이든 여성"처럼 여러 변환을 수행합니다.

GAN의 응용

  • 이미지 생성: 실제 사람처럼 보이는 사진 생성, 풍경 생성 등
  • 스타일 변환: 사진을 그림 같이 변환, 얼굴 스타일 변환
  • 데이터 증강: 부족한 학습 데이터를 GAN으로 생성하여 더 많은 데이터 확보
  • 이미지 복원: 흐릿한 이미지를 선명하게 복원, 노이즈 제거
  • 이미지 번역: 낮 사진을 밤 사진으로, 여름 사진을 겨울 사진으로 변환
  • 텍스트-이미지 생성: 텍스트 설명으로부터 이미지 생성 (DALL-E의 기초 기술)

GAN의 한계와 도전

모드 붕괴(Mode Collapse): 생성자가 다양한 이미지를 생성하지 못하고 비슷한 이미지만 반복적으로 생성하는 문제입니다. 마치 요리사가 같은 음식만 계속 만드는 것처럼 말입니다.
학습 불안정성(Training Instability): GAN의 학습은 매우 불안정할 수 있습니다. 생성자와 판별자가 균형을 맞추지 못하면, 학습이 발산하거나 수렴하지 않을 수 있습니다.
개선 방법: 최신 GAN 기술들은 이런 문제를 해결하기 위해 다양한 기법을 사용합니다. Wasserstein 거리, 스펙트럼 정규화(Spectral Normalization), 조건부 정규화(Conditional Normalization) 등이 있습니다.

2. 오토인코더와 VAE

오토인코더(Autoencoder, AE)의 개념

오토인코더(Autoencoder, AE)는 입력 데이터를 저차원의 잠재표현(latent representation)으로 압축했다가, 다시 원래 형태로 복원하는 신경망입니다. 인코더와 디코더 두 부분으로 구성됩니다.
입력 이미지 → 인코더 → 잠재표현 → 디코더 → 복원 이미지

인코더(Encoder)

인코더는 고차원의 입력 데이터(예: 784차원의 MNIST 이미지)를 저차원의 잠재표현(예: 32차원 벡터)으로 압축합니다. 이 과정에서 데이터의 가장 중요한 특징만 추출되어 저장됩니다.

잠재표현(Latent Representation)

잠재표현은 원본 데이터의 본질적인 정보를 담고 있는 저차원 벡터입니다. 예를 들어, 얼굴 이미지의 경우, 잠재표현은 "나이", "성별", "감정" 같은 개념적 특징을 인코딩할 수 있습니다.

디코더(Decoder)

디코더는 잠재표현을 입력으로 받아서 원래 형태의 데이터를 복원합니다. 이상적으로, 복원된 데이터는 원본 데이터와 최대한 유사해야 합니다.

오토인코더의 학습

오토인코더는 복원 오류를 최소화하도록 학습됩니다. 손실함수는 일반적으로 Mean Squared Error(MSE)입니다:

  • MSE = (원본 - 복원)²의 평균

오토인코더의 응용: 차원축소(Dimensionality Reduction)

오토인코더의 주요 응용은 차원축소입니다:

  • 데이터 압축: 이미지나 비디오를 더 작은 크기로 압축
  • 특성 추출: 원본 데이터에서 중요한 특성만 추출
  • 노이즈 제거: 손상된 입력으로부터 깨끗한 데이터 복원
  • 이상 탐지: 정상 데이터는 잘 복원하지만, 이상 데이터는 높은 복원 오류
  • 데이터 시각화: 2차원이나 3차원으로 축소하여 시각화

VAE(Variational Autoencoder)의 개념

VAE (Variational Autoencoder)는 오토인코더를 확률 모델로 확장한 것입니다. 잠재공간이 확률 분포를 따르도록 설계되어, 새로운 데이터를 생성할 수 있습니다.

VAE의 핵심: 확률적 잠재공간

기존 오토인코더에서는 잠재표현이 단순한 벡터이지만, VAE에서는 잠재표현이 확률분포(보통 정규분포)를 따릅니다:

비유로 이해하기: 기존 AE는 사진을 압축해서 보관하는 것이고, VAE는 사진의 "특징"을 확률분포로 보관하는 것입니다. VAE의 잠재공간에서는 인접한 영역들이 유사한 데이터를 나타내므로, 새로운 지점에서 샘플링하면 새로운 이미지를 생성할 수 있습니다.

VAE의 재매개변수화 트릭(Reparameterization Trick)

VAE의 인코더는 두 개의 출력을 생성합니다:

  • 평균(μ): 정규분포의 중심
  • 분산(σ): 정규분포의 퍼짐 정도

재매개변수화 트릭은 다음과 같이 작동합니다:

표준정규분포에서 ε 샘플링 → z = μ + σ × ε → 디코더에 입력

이 기법을 통해 역전파가 가능해지며, VAE를 end-to-end로 학습할 수 있습니다.

VAE의 손실함수

VAE의 손실함수는 두 부분으로 구성됩니다:

  • 복원 손실(Reconstruction Loss): 원본과 복원된 데이터의 차이
  • KL 발산(KL Divergence): 학습된 분포와 정규분포의 차이
총 손실 = 복원 손실 + β × KL 발산
이 균형이 중요합니다. KL 발산이 너무 크면 너무 흐릿한 이미지를 생성하고, 너무 작으면 단순 오토인코더처럼 작동합니다.
VAE 손실함수의 의미
복원 오차 (Reconstruction Error): 원본 이미지와 복원된 이미지의 차이를 최소화하므로, 모델이 입력 정보를 정확히 학습하도록 강제합니다
KL 발산 (KL Divergence): 잠재공간이 정규분포를 따르도록 제약합니다. 만약 KL 발산이 없다면 잠재공간이 불규칙해져서 새로운 샘플 생성이 불가능해집니다. 즉, 두 항의 균형이 생성 모델로 기능하게 합니다.

VAE의 응용

  • 이미지 생성: 잠재공간에서 샘플링하여 새로운 이미지 생성
  • 이미지 보간: 두 이미지의 잠재표현 사이를 보간하여 중간 형태의 이미지 생성
  • 이미지 편집: 잠재표현의 특정 차원을 조정하여 특정 속성 변경
  • 노이즈 제거: 손상된 데이터의 복원
  • 데이터 분석: 잠재공간 시각화를 통한 데이터 이해

AE vs VAE 비교

항목 오토인코더(AE) 변분 오토인코더(VAE)
잠재표현 결정적 벡터 확률분포(μ, σ)
생성 능력 생성 불가능 새 데이터 생성 가능
손실함수 복원 손실만 복원 손실 + KL 발산

복원 품질 일반적으로 더 선명 약간 흐림 (KL 손실 때문)

3. Diffusion 모델

Diffusion 모델의 개념

Diffusion 모델은 노이즈가 점진적으로 추가되는 "forward process"와 노이즈를 제거하는 "reverse process"로 구성된 생성 모델입니다. DDPM(Denoising Diffusion Probabilistic Model)이 대표적입니다.

Diffusion 모델은 근본적으로 다른 철학에 기반합니다: 복잡한 데이터 분포를 직접 학습하지 않고, 대신 노이즈 제거 과정을 학습합니다.

Forward Process: 노이즈 추가

Forward process는 원본 데이터에 점진적으로 노이즈를 추가하여 순수한 가우시안 노이즈가 될 때까지 진행합니다:

원본 이미지 x₀ → 약간의 노이즈 → 더 많은 노이즈 → 순수 노이즈 xₜ

이 과정은 다음과 같이 표현됩니다:

  • x₀: 원본 데이터
  • x₁, x₂, ..., xₜ: 점진적으로 노이즈가 추가된 중간 상태들
  • xₜ: T 스텝 후의 순수 가우시안 노이즈

Reverse Process: 노이즈 제거

Reverse process는 순수 노이즈로부터 시작하여 점진적으로 노이즈를 제거하여 원본 데이터와 유사한 샘플을 생성합니다:

순수 노이즈 xₜ → 노이즈 제거 → 더 많이 제거 → 생성된 이미지 x₀

Diffusion 모델의 학습

신경망은 forward process의 각 스텝에서 추가된 노이즈를 예측하도록 학습됩니다:

1
데이터 선택
학습 데이터 x₀를 선택합니다.
2
랜덤 시간선택
1~T 사이의 랜덤 스텝 t를 선택합니다.
3
노이즈 추가
xₜ에 노이즈를 추가합니다.
4
노이즈 예측
신경망이 추가된 노이즈를 예측합니다.

Stable Diffusion

Stable Diffusion은 Latent Diffusion Models을 기반으로 한 실용적인 이미지 생성 모델입니다. 이미지 공간 대신 압축된 잠재공간에서 diffusion을 수행하여 효율성을 크게 향상시켰습니다.

Stable Diffusion의 작동

  • 텍스트 인코딩: 텍스트 프롬프트를 임베딩으로 변환
  • 잠재공간 diffusion: 잠재공간에서 노이즈 제거 diffusion 수행
  • 이미지 복원: 디코더로 잠재표현을 이미지로 변환

Diffusion vs GAN vs VAE

항목 Diffusion GAN VAE
기본 원리 점진적 노이즈 제거 적대적 경쟁 확률적 인코딩
학습 안정성 매우 안정적 불안정함 안정적
생성 속도 느림 (수백 스텝) 빠름 (1스텝) 빠름 (1스텝)
생성 품질 매우 우수 우수 중간 (흐림)
가능도 추정 직접 계산 불가 불가능 가능
조건부 생성 용이 (클래스 레이블/텍스트) 가능 (conditional GAN) 가능

Diffusion 모델의 장점

  • 학습 안정성: 기존 생성 모델들보다 훨씬 안정적으로 학습됩니다.
  • 생성 품질: 현재 가장 높은 품질의 이미지를 생성합니다.
  • 다양성: 같은 입력에 대해 다양한 출력을 생성할 수 있습니다.
  • 조건부 제어: 텍스트, 이미지, 세그멘테이션 등으로 생성 과정을 조건화할 수 있습니다.

Diffusion 모델의 도전

느린 생성 속도: 고품질 생성을 위해 수백 개의 역전파 스텝이 필요하므로, 실시간 응용에는 부적합합니다.
개선 방법: DDIM(Denoising Diffusion Implicit Models)과 같은 가속화 기법들이 개발되었으며, 스텝 수를 10~50으로 줄일 수 있습니다.

4. 생성모델 비교 분석

세 모델의 특징 비유: GAN은 "즉흥 연주"처럼 빠르지만 불안정합니다. VAE는 "악보를 읽기"처럼 해석적이지만 다소 흐립니다. Diffusion은 "천천히 조각하기"처럼 느리지만 정교합니다.

생성모델 선택 가이드

고해상도 고품질 이미지가 필요하면: Diffusion을 선택하세요. 현재 최고의 품질을 제공합니다.
실시간 생성이 필요하면: GAN을 선택하세요. 가장 빠릅니다.
데이터 분석과 해석이 필요하면: VAE를 선택하세요. 가장 해석 가능합니다.
텍스트 제어 생성이 필요하면: Diffusion을 선택하세요. 조건화가 가장 용이합니다.

5. SNN: 생물학적 신경망

SNN이란 무엇인가?

SNN (Spiking Neural Network, 스파이킹 신경망)은 생물학적 뉴런의 작동 방식을 모사한 신경망입니다. 전통적인 인공신경망과 달리, SNN은 시간 정보를 활용하여 뉴런 간의 펄스(스파이크)를 통해 정보를 전달합니다.

우리의 뇌는 매우 효율적인 생물 컴퓨터입니다. 뇌의 신경세포들은 지속적으로 활성화되어 있지 않고, 필요한 순간에만 전기 신호(스파이크)를 보냅니다. SNN은 이런 생물학적 현실을 인공신경망으로 구현한 것입니다.

생물학적 뉴런의 작동 원리

  • 정보 수집: 수상돌기(dendrite)가 다른 뉴런으로부터 신호를 받습니다.
  • 신호 통합: 받은 신호들이 모여서 세포체(soma)에 축적됩니다.
  • 임계값 초과: 축적된 신호가 특정 수준(임계값)을 넘으면, 뉴런이 "발화(spike)"합니다.
  • 신호 전송: 축삭(axon)을 통해 전기 신호가 다른 뉴런들에게 전달됩니다.
  • 재설정: 발화 후, 뉴런은 불응기(refractory period)를 거쳐 원래 상태로 돌아갑니다.

전통적 ANN과 SNN의 차이

신호 전달 방식의 비유: 전통적인 ANN은 "밝기 조절이 되는 전등"과 같습니다. 신경세포가 항상 어느 정도의 신호를 내보내는데, 그 강도만 조절되는 것입니다. 반면 SNN은 "모스 부호"와 같습니다. 신경세포는 "신호를 보낸다"(1) 또는 "보내지 않는다"(0)의 이진 선택만 하지만, 신호를 보내는 시점(타이밍)이 정보를 담고 있습니다.
항목 전통적 ANN SNN
신호 형태 연속값 (0~1) 이산값 (0 또는 1)
시간 개념 없음 있음 (타이밍 중요)
에너지 소비 높음 매우 낮음
모델 비생물학적 생물학적
호환 칩 CPU, GPU 뉴로모픽 칩

SNN의 핵심 개념

막전위(Membrane Potential)

실제 뉴런과 마찬가지로, SNN의 인공 뉴런도 막전위라는 개념을 가집니다. 이것은 뉴런이 스파이크를 발화할 때까지 축적되는 "에너지" 같은 것입니다. 들어오는 신호들이 막전위를 높이고, 시간이 지나면서 자연적으로 감소합니다.

누적-발화 모델(Leaky Integrate-and-Fire, LIF)

이는 SNN에서 가장 많이 사용되는 뉴런 모델입니다:

  • 누적(Integrate): 들어오는 신호를 모습니다.
  • 발화(Fire): 임계값을 넘으면 스파이크를 보냅니다.
  • 누출(Leak): 시간이 지나면서 축적된 신호가 조금씩 감소합니다.

스파이크 타이밍 의존적 가소성(STDP)

생물학적 뉴런의 학습 메커니즘입니다: "선행-후행 학습"이라고도 불리며, 만약 A 뉴런의 스파이크가 B 뉴런의 스파이크보다 조금 먼저 일어나면, A→B 연결이 강해집니다. 반대로 나중에 일어나면, 연결이 약해집니다.

SNN의 장점

  • 에너지 효율성: 필요한 순간에만 계산하므로 전력 소비가 매우 적습니다. 임베디드 시스템이나 엣지 디바이스에 매우 유용합니다.
  • 시간적 패턴 인식: 시간 정보를 직접 활용하므로, 시계열 데이터나 시간적 패턴을 매우 잘 인식합니다.
  • 신경형 하드웨어 호환성: 생물학적 원리에 기반하므로, 신경형 칩(neuromorphic chips)과 자연스럽게 맞아떨어집니다.
  • 비동기 처리: 모든 연산이 동시에 일어날 필요가 없어서, 병렬 처리가 용이합니다.

뉴로모픽 칩(Neuromorphic Chips)

뉴로모픽 칩: SNN을 하드웨어 수준에서 구현한 칩입니다. Intel의 Loihi 칩, IBM의 TrueNorth 같은 신경형 컴퓨터들이 SNN을 기본으로 설계되었습니다. 이들은 기존 CPU/GPU보다 훨씬 에너지 효율적입니다.

SNN의 응용

  • 로봇 제어: 로봇이 환경을 인식하고 반응하는 데 매우 효율적입니다.
  • 에지 AI: 스마트폰이나 IoT 기기 같은 제한된 전력의 장치에서 AI를 구동합니다.
  • 시각 처리: 동적 시각 센서(DVS)와 결합하여 매우 빠른 객체 인식을 합니다.
  • 스피치 인식: 음성 신호의 시간적 패턴을 잘 포착합니다.
  • 이상 탐지: 비정상적인 패턴을 실시간으로 감지합니다.
  • 우주선: 높은 방사선 환경에서도 작동 가능합니다.

SNN의 도전 과제

학습의 어려움: SNN은 스파이크가 미분 불가능하기 때문에, 전통적인 역전파(backpropagation)를 직접 사용할 수 없습니다. 이를 극복하기 위해 "Surrogate Gradient Method", "ANN-SNN 변환", "신경 진화" 같은 새로운 기법들이 개발되고 있습니다.
훈련 속도: SNN의 학습은 기존 신경망보다 느릴 수 있습니다. 여러 시간 스텝에 걸쳐 시뮬레이션해야 하기 때문입니다.
최근 진전: 2022년 이후, SNN 성능이 급속도로 개선되고 있습니다. Surrogate Gradient Method 같은 기법 덕분에, 대규모 이미지 데이터셋에서도 전통적 신경망과 경쟁할 수 있는 수준에 도달했습니다.

6. GNN: 그래프 신경망

GNN이란 무엇인가?

GNN (Graph Neural Network, 그래프 신경망)은 그래프 구조의 데이터를 학습하기 위해 설계된 신경망입니다. 노드(점)와 엣지(선)로 이루어진 네트워크 데이터를 효과적으로 처리합니다.

그래프 데이터의 구성

  • 노드(Node): 그래프의 개별 점들입니다. 소셜 네트워크에서는 사람, 분자 구조에서는 원자가 노드입니다.
  • 엣지(Edge): 노드들을 연결하는 선들입니다. 소셜 네트워크에서는 친구 관계, 분자 구조에서는 화학 결합이 엣지입니다.
  • 노드 특성(Node Features): 각 노드가 가진 정보입니다. 사람 노드는 나이, 성별 같은 속성을 가질 수 있습니다.
  • 엣지 특성(Edge Features): 각 엣지가 가진 정보입니다. 거리, 강도, 유형 등을 나타낼 수 있습니다.

그래프 데이터의 예시

  • 소셜 네트워크: 노드 = 사람, 엣지 = 친구 관계
  • 분자 구조: 노드 = 원자, 엣지 = 화학 결합
  • 지식 그래프: 노드 = 개념, 엣지 = 관계 (예: "서울" -"수도"- "대한민국")
  • 교통 네트워크: 노드 = 도시, 엣지 = 도로 또는 항공노선
  • 추천 시스템: 노드 = 사용자와 상품, 엣지 = 구매 또는 평가
  • 단백질 상호작용: 노드 = 단백질, 엣지 = 상호작용

GNN의 핵심: 메시지 전달(Message Passing)

메시지 전달은 GNN의 가장 중요한 아이디어입니다. 각 노드가 이웃 노드들로부터 정보(메시지)를 받고, 이를 처리해서 자신의 상태를 업데이트합니다.
이웃 정보 수집 → 메시지 생성 → 집계 → 노드 업데이트
메시지 전달의 비유: SNS에서 정보가 퍼지는 방식을 생각해보세요. 한 사람이 소식을 올리면, 그 사람의 친구들이 그것을 봅니다. 그 친구들의 친구들도 보게 되고, 점점 퍼져 나갑니다. GNN에서 각 노드는 자신과 연결된 이웃 노드들의 정보를 모아서 자신의 상태를 업데이트하는, 마치 이 전파 메커니즘과 같은 방식으로 작동합니다.

GNN의 주요 유형

GCN (Graph Convolutional Network): 이미지 처리의 합성곱(CNN)을 그래프로 확장한 것입니다. 각 노드의 이웃 노드들의 정보를 가중 평균 방식으로 통합합니다.
GAT (Graph Attention Network): 어떤 이웃이 더 중요한지를 학습합니다. 모든 이웃을 동등하게 취급하지 않고, "주의(attention)" 메커니즘을 사용해서 중요한 이웃에 더 많은 가중치를 부여합니다.
GraphSAGE (Graph Sample and Aggregate): 매우 큰 그래프에서 효율적으로 작동합니다. 모든 이웃을 고려하는 대신, 일부 이웃을 샘플해서 집계합니다. 확장성이 뛰어납니다.

GNN의 응용

추천 시스템

사용자-상품 그래프를 만들어서, 사용자와 상품 사이의 복잡한 관계를 학습합니다. 이를 통해 어떤 사용자에게 어떤 상품을 추천할 것인가를 결정합니다.

약물 발견

분자 구조를 그래프로 표현한 후, GNN을 사용해서 분자의 특성을 예측합니다. 예: "이 분자는 암 치료에 효과가 있을까?"

사기 탐지

거래 네트워크(계좌 간의 송금 관계)를 그래프로 표현한 후, 비정상적인 패턴을 찾아냅니다.

교통 예측

도시의 교통 네트워크를 그래프로 표현한 후, "내일 이 도로의 교통량은 얼마나 될 것인가?"를 예측합니다.

기타 응용

  • 단백질 구조 예측 및 상호작용 예측
  • 텍스트 분류 및 자연어 처리
  • 커뮤니티 탐지(그래프에서 밀집된 부분군 찾기)
  • 링크 예측(새로운 엣지가 생길 확률 예측)
  • 지식 그래프 완성
  • 소셜 네트워크 분석

GNN과 LLM: 환각 완화

최근 주목받는 응용이 바로 GNN을 사용한 LLM(대규모 언어모델) 환각 완화입니다. ChatGPT 같은 LLM은 때때로 존재하지 않는 정보를 마치 진실인 것처럼 생성합니다. 이를 "환각(hallucination)"이라고 합니다.

해결책: 지식 그래프(노드: 개념, 엣지: 관계)와 GNN을 결합하면, LLM의 출력을 지식 그래프에 대비해서 검증할 수 있습니다. 이렇게 하면, LLM이 생성한 답이 실제 사실에 기반한 것인지 확인할 수 있습니다.

GNN의 장점

  • 구조 정보 활용: 데이터의 관계 구조를 직접 활용할 수 있습니다.
  • 확장성: GraphSAGE 같은 기법으로 매우 큰 그래프도 처리할 수 있습니다.
  • 해석 가능성: 어떤 이웃의 정보가 더 중요했는지 알 수 있어서, 모델의 결정을 이해할 수 있습니다.
  • 일반화 능력: 유사한 구조를 가진 다른 그래프에도 적용할 수 있습니다.

GNN의 도전 과제

과다-평활화(Over-smoothing): GNN 층이 깊어질수록, 모든 노드가 비슷한 표현을 가지게 되는 문제가 발생합니다. 마치 전화 게임에서 메시지가 왜곡되듯이, 정보가 여러 번 전파되면서 개성을 잃게 되는 것입니다.
확장성 문제: 엣지가 매우 많은 그래프(밀집 그래프)는 여전히 처리하기 어렵습니다.
연구 진행 중: DropMessage, JumpingKnowledge, GraphGym 같은 새로운 기법들이 계속 개발되고 있습니다.

7. 전이학습과 도메인 적응

전이학습(Transfer Learning)의 개념

전이학습(Transfer Learning)은 한 분야에서 학습한 지식을 다른 분야에 활용하는 기법입니다. "사전학습된 모델"에서 배운 특성들을 활용하여, 새로운 작업을 더 효율적으로 학습하는 것입니다.

전이학습의 핵심 아이디어는 간단합니다: 처음부터 모든 것을 배우지 말고, 이미 배워진 지식을 활용하세요. 마치 새로운 언어를 배울 때, 모국어의 문법 개념을 활용하는 것처럼 말입니다.

전이학습의 작동 원리

사전학습 → 특성 추출 → 파인튜닝 → 새 작업 완료

사전학습(Pre-training)

대규모 데이터셋(예: ImageNet, BERT 코퍼스)으로 모델을 미리 학습합니다. 이 과정에서 모델은 일반적인 특성들(예: 이미지의 엣지, 텍스트의 문법)을 학습합니다.

파인튜닝(Fine-tuning)

사전학습된 모델의 가중치를 초기값으로 사용하고, 새로운 작업의 데이터로 적응시킵니다. 보통 낮은 학습률을 사용하여 이미 배운 특성을 유지하면서 조정합니다.

1
사전학습 모델
대규모 데이터로 학습된 모델 로드
2
구조 수정
마지막 층을 새 작업에 맞게 수정
3
파인튜닝
새 데이터로 적응시키기
4
평가
새 작업에서의 성능 확인

전이학습의 전략

특성 추출(Feature Extraction)

사전학습된 모델의 가중치를 고정하고, 마지막 분류 층만 새 작업에 맞게 학습합니다. 데이터가 적을 때 유용합니다.

완전 파인튜닝(Full Fine-tuning)

모든 가중치를 학습 가능하게 하고, 전체 모델을 새 데이터에 적응시킵니다. 데이터가 충분할 때 더 좋은 결과를 줄 수 있습니다.

부분 파인튜닝(Partial Fine-tuning)

모델의 일부 층만 학습하고 나머지는 고정합니다. 계산 효율성과 성능의 균형을 맞춥니다.

전이학습의 응용

  • 컴퓨터 비전: ImageNet으로 사전학습된 ResNet을 의료 이미지 분류에 사용
  • 자연어 처리: BERT, GPT 같은 사전학습 모델을 감정 분류, 기계 번역 등에 활용
  • 음성 인식: 대규모 음성 데이터로 사전학습된 모델을 특정 언어나 악센트에 적응
  • 추천 시스템: 일반적인 사용자-상품 관계로 사전학습하고, 특정 플랫폼에 파인튜닝

도메인 적응(Domain Adaptation)

도메인 적응(Domain Adaptation)은 전이학습의 특수한 경우입니다. 원본 도메인(source domain)에서 학습한 모델을 대상 도메인(target domain)에 적응시키는 기법입니다.

도메인 시프트(Domain Shift)

현실 세계에서는 학습 데이터와 실제 사용 데이터가 다를 수 있습니다. 예를 들어:

  • 광도 변화: 실험실 조명 아래서 학습했지만, 야외에서 사용
  • 카메라 변화: 한 종류의 카메라로 학습했지만, 다른 카메라 사용
  • 데이터 분포 변화: 젊은 사람의 얼굴로 학습했지만, 나이 든 사람의 얼굴 인식

도메인 적응 기법

비지도 도메인 적응(Unsupervised Domain Adaptation)

대상 도메인의 레이블이 없을 때 사용됩니다. 원본과 대상 도메인의 특성 분포를 유사하게 만드는 방법들:

  • 최대 평균 불일치(MMD): 두 도메인의 평균 특성을 가깝게 만들기
  • 적대적 도메인 적응: GAN 기법을 사용하여 도메인 구분 불가능하게 만들기
  • 자기 학습(Self-training): 모델이 대상 도메인에서 신뢰도 높은 예측을 레이블로 사용

준지도 도메인 적응(Semi-supervised Domain Adaptation)

대상 도메인의 일부 레이블이 있을 때 사용됩니다. 레이블된 샘플로 직접 학습하고, 레이블되지 않은 샘플로는 도메인 적응을 수행합니다.

전이학습의 장점과 한계

장점:
  • 필요한 학습 데이터 양 감소
  • 학습 시간 단축
  • 작은 데이터셋에서도 좋은 성능
  • 모델의 일반화 능력 향상
한계: 원본 도메인과 대상 도메인이 너무 다르면 오히려 성능이 저하될 수 있습니다(음의 전이). 또한 도메인 시프트가 크면 적응이 어려울 수 있습니다.

상황별 모델 선택 가이드

생성 모델 선택 가이드 (종합)
이미지 생성 (신규): GAN (고품질, 빠른 생성) 또는 Diffusion 모델 (최고 품질, 느림)
데이터 증강: VAE (다양한 변형 생성 가능)
텍스트 생성: Transformer 기반 모델 (GPT, T5 등)
그래프/관계 데이터: GNN (Graph Neural Network, 구조 학습)
에너지 효율: SNN (Spiking Neural Network, IoT/엣지 기기)
새로운 이미지를 생성하고 싶을 때:
  • 최고 품질: Diffusion (Stable Diffusion, DALL-E)
  • 실시간 필요: GAN
  • 해석 가능: VAE
이미지를 변환하고 싶을 때:
  • 스타일 변환: StyleGAN, CycleGAN
  • 보간/편집: VAE
  • 쌍 기반 변환: Pix2Pix
에너지 효율이 중요할 때:
  • 로봇/IoT: SNN
  • 엣지 AI: SNN + 양자화
  • 뉴로모픽 칩: SNN
관계 데이터를 분석하고 싶을 때:
  • 추천: GNN
  • 약물 발견: GNN
  • 사기 탐지: GNN
  • 지식 그래프: GNN
데이터가 부족할 때:
  • 먼저 시도: 전이학습
  • 데이터 증강: GAN
  • 보조 타스크: 멀티태스크 학습

학습 곡선 및 성능 비교

일반적인 추세:

  • 작은 데이터: 전이학습 > Diffusion > GAN > 처음부터 학습
  • 중간 데이터: 전이학습 ≈ Diffusion > GAN > 처음부터
  • 큰 데이터: 모두 비슷한 성능 (전이학습이 여전히 약간 앞)
  • 실시간 필요: GAN > SNN > Diffusion
  • 품질 중심: Diffusion > GAN > VAE > 전통 모델

미래의 방향

다중 모델 조합: 최신 AI 시스템들은 여러 모델을 조합하여 사용합니다.
  • Diffusion + GNN: 조건화된 생성
  • Transfer Learning + Diffusion: 도메인 적응 생성
  • SNN + GNN: 효율적 그래프 처리

이 장의 핵심 정리

  • GAN은 빠른 생성: 생성자와 판별자의 경쟁을 통해 새로운 데이터를 생성하지만, 학습이 불안정할 수 있습니다.
  • VAE는 해석 가능: 확률적 잠재공간을 통해 데이터를 압축하고 생성하며, 가장 해석 가능합니다.
  • Diffusion은 최고 품질: 점진적 노이즈 제거로 가장 높은 품질의 이미지를 생성하지만 느립니다.
  • SNN은 매우 효율적: 생물학적 뉴런을 모사하여 극도로 낮은 에너지 소비로 작동합니다.
  • GNN은 관계 전문: 그래프 구조의 데이터에서 복잡한 관계를 효과적으로 학습합니다.
  • 전이학습은 필수: 데이터가 부족한 대부분의 실제 문제에서 가장 실용적인 접근법입니다.
  • 모델 선택이 중요: 문제의 특성, 데이터 양, 계산 자원에 따라 최적의 모델을 선택해야 합니다.
