1. Seq2Seq (시퀀스 투 시퀀스)

기본 개념

Seq2Seq는 "입력 문장을 출력 문장으로 변환"하는 신경망 구조입니다. 기계번역(영어→한국어), 챗봇, 요약 같은 작업에 사용됩니다.

Seq2Seq: 가변 길이의 입력 시퀀스를 가변 길이의 출력 시퀀스로 매핑하는 신경망 모델. 인코더와 디코더 두 부분으로 구성됩니다.

인코더-디코더 구조

Seq2Seq는 두 부분으로 나뉩니다:

  • Encoder: 입력 문장을 읽고 압축하여 "핵심 정보"를 고정 길이 벡터(Context Vector)로 만듭니다
  • Decoder: 그 벡터를 받아 출력 문장을 한 단어씩 생성합니다

Encoder 상세 동작

Encoder의 작동 과정:

1. 입력 처리: 입력 문장의 각 단어를 임베딩으로 변환
2. RNN 순차 처리: LSTM 또는 GRU가 한 단어씩 읽으며 상태(hidden state)를 업데이트
3. 최종 상태 수집: 마지막 단어까지 읽은 후의 hidden state가 Context Vector가 됨
4. 정보 압축: 긴 문장의 모든 정보를 고정 길이 벡터에 압축

Decoder 상세 동작

Decoder의 작동 과정:

1. 초기 상태: Encoder의 Context Vector로 Decoder의 초기 hidden state 설정
2. 시작 토큰: [START] 토큰을 첫 입력으로 사용
3. 순차 생성: 각 타임스텝에서 다음 단어를 생성
4. 종료 조건: [END] 토큰이 나올 때까지 반복

Teacher Forcing (교사 강제)

디코더 학습 시 사용하는 기법:

  • 개념: 학습 중에는 모델의 예측이 틀렸을 때도 "올바른 답변"을 다음 입력으로 제공
  • 예시: 모델이 "hello"를 잘못 예측했으나, 다음 입력으로 정답인 "안녕"을 강제로 제공
  • 효과: 학습이 빠르고 안정적
  • 문제: 추론 시 모델은 자신의 실수를 수정할 기회가 없어 오류 누적 발생
  • 해결: Curriculum Learning이나 Scheduled Sampling으로 완화
입력: "Hello"
→
Encoder 압축
→
Context Vector
→
Decoder 전개
→
출력: "안녕"

한계점: 정보 손실

고정 길이 벡터의 문제점:

  • 긴 문장을 고정 길이 벡터로 압축하면 정보가 손실됩니다
  • 100단어짜리 문장을 128개 숫자로 압축하면 세부 정보가 사라집니다
  • 문장이 길수록 번역 품질이 저하됩니다
  • 기울기 소실(Vanishing Gradient) 문제 발생
유추: 통역사가 10분짜리 회의 전체를 한 줄 메모에 압축한 후 번역하려고 할 때, 중요한 세부사항이 빠질 수 있는 것과 같습니다.
알아두기: 이러한 한계를 극복하기 위해 "Attention" 메커니즘이 등장했습니다.

2. Attention 메커니즘

왜 필요한가?

Seq2Seq의 고정 길이 벡터 한계를 극복하기 위해 Attention이 등장했습니다. Attention의 핵심 아이디어는 간단합니다:

Attention의 핵심: 출력할 때마다 입력의 모든 부분을 다시 살펴보되, 현재 생성할 단어와 "관련 있는" 입력 부분에 더 많은 주의(높은 가중치)를 기울인다.

작동 방식: Query-Key-Value

Attention은 세 가지 요소로 작동합니다:

  • Query (Q): "내가 어떤 정보가 필요한가?" - 현재 디코더의 상태
  • Key (K): "각 입력 부분은 어떤 특징을 가지는가?" - 입력의 특징
  • Value (V): "실제 정보의 내용은 무엇인가?" - 입력의 실제 데이터

어텐션 스코어 계산 상세

Attention의 계산 과정:

1단계: 유사도 계산 (Dot Product)
- Query와 Key를 곱해서 유사도를 계산합니다
- Similarity = Q · K^T

2단계: 정규화 (Softmax)
- 유사도를 확률로 변환합니다 (0~1 범위)
- Attention Weight = Softmax(Similarity)

3단계: 가중합 (Weighted Sum)
- Value를 가중평균으로 합칩니다
- Output = Attention Weight × V

구체적 계산 예시

"나는 밥을 먹었다"에서 "먹었다"를 생성할 때:

Query: 현재 생성 중인 단어 "먹었다"의 상태 벡터
Key, Value: 입력의 모든 단어들

계산:
Scores = [Query · "나" Key, Query · "는" Key, Query · "밥을" Key, Query · "먹었다" Key]
→ [0.2, 0.1, 0.6, 0.1] (정규화 전)
Weights = Softmax([0.2, 0.1, 0.6, 0.1])
→ [0.15, 0.10, 0.65, 0.10] (합=1.0)
Output = 0.15·"나"Value + 0.10·"는"Value + 0.65·"밥을"Value + 0.10·"먹었다"Value
도서관 비유: Query는 '찾고 싶은 주제', Key는 '각 책의 제목', Value는 '책의 내용'. Q와 K의 유사도가 높은 책(Attention Weight)의 Value를 더 많이 참고합니다.

Bahdanau vs Luong Attention 비교

Attention 메커니즘의 두 가지 주요 변형:

특징 Bahdanau Attention Luong Attention 제안 시기 2015년 (Bahdanau et al.) 2015년 (Luong et al.) - 동시기 스코어 함수 score = v^T tanh(W[h_t; h_s]) score = h_t^T W h_s 계산 복잡도 더 복잡 (tanh 활성화 필요) 더 간단 (내적만) 정렬 메커니즘 Concat 기반 (결합) Dot-product 기반 (내적) 성능 약간 더 정확 (추가 파라미터) 간단하면서도 효과적 주로 사용 Seq2Seq 초기 모델 현대 Transformer 모델 계산식 예 비선형 변환으로 더 표현력 있음 선형으로 더 효율적

효과와 의미

Attention 메커니즘으로 인해:

  • 긴 문장도 효율적으로 처리 가능
  • 어떤 입력 부분이 각 출력에 영향을 미쳤는지 해석 가능
  • 번역 품질 대폭 향상
  • 기울기가 모든 입력으로 역전파되어 장거리 의존성 문제 해결
유추: 시험을 볼 때 교과서 전체를 외우는 대신, 현재 문제와 관련된 페이지를 펼쳐보고 그 부분을 집중해서 읽는 것과 같습니다.
다양한 Attention: Bahdanau Attention, Luong Attention 등 여러 변형이 존재합니다.

3. Self-Attention과 Multi-Head Attention

Self-Attention: 자기 참조

Self-Attention은 Attention을 한 단계 더 발전시킨 것입니다. 입력 문장 내의 단어들이 서로 어떤 관계를 가지는지를 파악합니다.

Self-Attention: 문장 내의 모든 단어가 문장 내의 다른 모든 단어와의 관계를 학습하는 메커니즘. Query, Key, Value가 모두 같은 입력에서 나옵니다.

문맥 파악의 핵심

자기참조의 예시:

  • 문장: "그것은 매우 맛있다"
  • "그것"이 무엇을 가리키는지 파악하려면, "그것"을 앞의 단어들과 비교
  • "밥", "국", "고기" 중 가장 유사한 것을 찾음
  • 이 과정이 자동으로 학습됩니다
유추: 글을 읽을 때 대명사가 무엇을 가리키는지 파악하기 위해 앞뒤 문맥을 보는 것처럼, 모델이 문장 내 단어 간의 관계를 자동으로 파악합니다.

Multi-Head Attention: 다양한 관점

Multi-Head Attention은 동일한 입력을 다양한 관점에서 동시에 분석합니다:

Head 분석 관점 예시 Head 1 문장의 문법 구조 주어-동사-목적어 관계 Head 2 단어의 의미 관계 유의어, 반의어 관계 Head 3 대명사의 선행사 "그것"이 무엇을 가리키는지 Head 4 시간적 관계 "이전", "이후" 같은 시간 표현

Self-Attention QKV 계산 과정 (Step-by-step)

Self-Attention의 세 단계 상세:

1
Query 생성
입력을 Query 행렬 Q로 변환
Q = Input × W^Q
"각 단어가 다른 단어의 어느 부분을 봐야하는가?"
2
Key 생성
입력을 Key 행렬 K로 변환
K = Input × W^K
"각 단어가 어떤 특징을 가지는가?"
3
Value 생성
입력을 Value 행렬 V로 변환
V = Input × W^V
"각 단어의 실제 콘텐츠는 무엇인가?"
4
유사도 계산
Query와 Key의 내적
Scores = Q × K^T
행렬 크기: [seq_len × seq_len]
5
스케일 정규화
복잡도로 정규화
Scaled = Scores / √d_k
d_k = embedding_dim/num_heads
6
소프트맥스
확률로 변환
Weights = Softmax(Scaled)
각 행의 합 = 1.0
7
가중합
Value의 가중평균
Output = Weights × V
최종 크기: [seq_len × d_v]

계산 복잡도 분석

Self-Attention의 시간 복잡도:

  • Attention 계산: Q × K^T 연산 = O(n²) (n = 시퀀스 길이)
  • 가중합 계산: Weights × V 연산 = O(n²)
  • 전체 복잡도: O(n²)
  • 공간 복잡도: O(n²) - Attention 가중치 행렬 저장 필요
  • 의미: 시퀀스가 2배 길어지면 계산량은 4배 증가
  • 한계: 매우 긴 문서(수만 단어)는 계산 불가능
  • 개선안: Sparse Attention, Linear Attention 등으로 O(n)으로 단축 시도 중

작동 원리

Multi-Head Attention의 과정:

1단계: 입력 분할
- 입력을 여러 개의 서브스페이스로 선형 변환 (h개의 head로)
- 각 head는 embedding_dim의 1/h 크기로 축소

2단계: 각 Head에서 Self-Attention 계산
- h개의 Self-Attention을 병렬로 계산
- 각 Head는 다른 가중치 사용: W^Q_i, W^K_i, W^V_i
- MultiHead Attention = [head_1 | head_2 | ... | head_h]

3단계: 결과 연결 및 변환
- 모든 Head의 결과를 이어붙임 (concatenate)
- 최종 선형 변환으로 통합: Output = Concat(head_1, ...) × W^O
- W^O로 다시 원래 차원으로 복원
효과: 다중 관점으로 분석하므로 더욱 정확한 의미 파악이 가능합니다.

4. Transformer 아키텍처

Transformer 핵심: RNN의 순차 처리를 Self-Attention의 병렬 처리로 대체. 모든 위치가 동시에 모든 다른 위치를 참조 가능 → 학습 속도 대폭 향상

혁신적인 구조

Transformer는 2017년 "Attention Is All You Need" 논문에서 제안되었습니다. RNN(순환 신경망)을 완전히 대체했으며, Attention만으로 이루어진 혁신적인 구조입니다.

Transformer: Self-Attention과 Multi-Head Attention만을 사용하여 병렬 처리가 가능한 신경망 구조. Encoder와 Decoder 스택으로 구성됩니다.
중요: Transformer는 이후 모든 대형 언어 모델(GPT, BERT, Claude 등)의 기반이 됩니다.

Encoder: 입력 처리 상세

Encoder는 입력 문장을 깊이 있게 분석합니다. 여러 층(6층)으로 쌓여 있으며, 각 층은:

  • Multi-Head Self-Attention: 입력 내 단어들 간의 관계 파악
  • Feed Forward Network (FFN): 각 단어를 독립적으로 더 풍부하게 변환
  • Residual Connection: 이전 층의 출력을 더해주는 건너뛰기 연결
  • Layer Normalization: 수치 안정성을 위한 정규화

Residual Connection (잔여 연결)

각 서브층의 출력이 입력과 더해지는 구조:

Output = SubLayer(Input) + Input

효과:
- 깊은 네트워크에서 기울기가 잘 역전파됨
- 각 층이 작은 변화만 학습하면 됨
- 더 깊은 모델 학습 가능 (100+층)
- 모델 안정성 향상

Layer Normalization (층 정규화)

배치 정규화 대신 사용하는 정규화 기법:

  • 정의: 각 샘플에서 피처 차원을 정규화 (배치 차원 아님)
  • 장점: 배치 크기에 의존하지 않음, RNN에 효과적
  • 적용: Attention 후, FFN 후에 적용
  • 수식: LN(x) = γ * (x - μ) / √(σ² + ε) + β
  • 결과: 각 층의 입력/출력을 일정한 범위로 유지

Feed Forward Network (FFN)

각 위치마다 동일하게 적용되는 완전연결 신경망:

FFN(x) = max(0, x·W₁ + b₁)·W₂ + b₂

구조:
Input(d_model) → Dense(d_ff) → ReLU → Dense(d_model)
보통 d_ff = 4 × d_model (예: 512 → 2048 → 512)

효과:
- 각 단어의 특성을 비선형으로 강화
- 다른 단어와 무관하게 독립적으로 처리
- 모델의 비선형 표현력 증가

Decoder: 출력 생성

Decoder는 한 번에 한 단어씩 생성합니다:

  • Masked Self-Attention: 자신 앞의 단어들만 볼 수 있도록 제한 (미래 보기 방지)
  • Encoder-Decoder Attention: Encoder의 출력(Key, Value)을 참고하여 단어 생성
  • Feed Forward Network: 생성된 단어를 더 풍부하게 표현

Positional Encoding: 위치 정보

Transformer는 병렬 처리를 하기 때문에, 단어의 순서를 명시적으로 알려줘야 합니다:

Positional Encoding: 사인(sin)과 코사인(cos) 함수를 사용하여 각 단어의 위치 정보를 인코딩합니다. 이를 통해 모델이 단어의 위치를 인식합니다.

Positional Encoding 수식 상세

각 타임스텝(위치) pos와 차원 i에 대한 Positional Encoding:

PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

매개변수 설명:
- pos: 단어의 위치 (0, 1, 2, ...)
- i: 차원 인덱스 (0, 1, 2, ..., d_model/2)
- d_model: 모델의 임베딩 차원 (보통 512)
- 10000: 주파수 스케일링 상수 (다양한 주파수로 위치 인코딩)

Positional Encoding의 특징

  • 사인/코사인 선택: 짝수 차원은 sin, 홀수 차원은 cos 사용
  • 주파수 감소: 저주파(왼쪽 차원)는 천천히 변하고, 고주파(오른쪽 차원)는 빠르게 변함
  • 상대적 위치: PE는 상대적 위치 관계를 자동으로 학습 가능
  • 학습 불가: 일반적으로 고정된 값 (학습하지 않는 버전도 있음)
  • 덧셈: 위치 인코딩은 임베딩에 더해짐: Embedding + PE

구체적 예시

문장 "나는 밥을 먹었다" (d_model=512)에서:

pos=0 (나): PE(0, 0) = sin(0), PE(0, 1) = cos(0), ...
pos=1 (는): PE(1, 0) = sin(1/10000^0), PE(1, 1) = cos(1/10000^0), ...
pos=2 (밥을): PE(2, 0) = sin(2/10000^0), PE(2, 1) = cos(2/10000^0), ...
pos=3 (먹었다): PE(3, 0) = sin(3/10000^0), PE(3, 1) = cos(3/10000^0), ...

각 위치는 고유한 PE 벡터 → 모델이 순서를 인식

기존 RNN과의 비교

측면 RNN Transformer 처리 방식 순차적 (한 단어씩) 병렬 처리 (모든 단어 동시) 학습 속도 느림 매우 빠름 긴 문장 처리 정보 손실 많음 정보 손실 적음 장거리 의존성 어려움 (기울기 소실) 매우 용이 (직접 연결) 메모리 사용 메모리 효율적 더 많은 메모리 필요 계산 복잡도 O(n) O(n²)

Transformer의 강점

  • 병렬 처리로 큰 모델을 빠르게 학습 가능
  • 긴 문장의 의미를 정확히 포착
  • 전이학습(Transfer Learning)이 매우 효과적
  • 멀티헤드 어텐션으로 다양한 관계 동시 포착
  • 병목 현상 없음 (RNN의 순차 처리 제거)
구조: Encoder는 문맥 이해, Decoder는 텍스트 생성에 최적화되어 있습니다.

5. BERT 상세

BERT의 기본 개념

BERT는 2018년 Google에서 발표한 "Bidirectional Encoder Representations from Transformers"의 약자입니다. Transformer의 Encoder를 사용합니다.

BERT: 양방향(양쪽 모두 읽음) 문맥 이해에 최적화된 사전학습 모델. 텍스트의 의미를 매우 정확히 이해합니다.

BERT의 사전학습 방식 상세

BERT는 두 가지 자기지도학습(Self-Supervised Learning) 목표로 학습됩니다:

1. MLM (Masked Language Model) 상세

마스킹된 언어 모델 학습:

  • 원리: 문장의 일부 단어를 [MASK] 토큰으로 가림
  • 확률: 랜덤하게 15%의 단어를 마스킹
  • 목표: 앞뒤 문맥으로 가려진 단어를 맞추기
  • 예시 1: "나는 [MASK] 먹었다" → "밥" 예측
  • 예시 2: "영화는 [MASK] 재미있다" → "매우" 또는 "정말" 예측
  • 효과: 양방향 문맥 이해 (좌측과 우측 모두 볼 수 있음)
  • 특징: 다의성 처리 - 같은 단어도 문맥에 따라 다른 임베딩

2. NSP (Next Sentence Prediction) 상세

문장 관계 예측 학습:

  • 원리: 두 문장이 실제로 연속인지 여부를 판단 (이진 분류)
  • 입력: "[CLS] 문장1 [SEP] 문장2 [SEP]"
  • 목표: IsNext 또는 NotNext 예측
  • 예시 IsNext: "나는 밥을 먹었다" + "맛있었다"
  • 예시 NotNext: "나는 밥을 먹었다" + "고양이가 잔다"
  • 효과: 문장 간의 의미 관계 학습
  • 활용: 자연어 추론, 문장 유사도 판정에 도움

MLM과 NSP 동시 학습

사전학습 손실함수:

Total Loss = MLM Loss + NSP Loss

MLM Loss: 마스킹된 단어 예측 오차 (크로스 엔트로피)
NSP Loss: 문장 관계 예측 오차 (이진 크로스 엔트로피)

두 목표가 함께 작동하여 더 강력한 문맥 표현 학습

BERT의 특징

특징 설명 양방향성 앞의 단어와 뒤의 단어를 모두 고려하여 단어 의미 파악 문맥 의존성 같은 단어도 문맥에 따라 다른 의미 표현 (Contextual Embedding) 문장 이해 감정분석, 질의응답, 텍스트 분류에 탁월 텍스트 생성 Encoder만 있으므로 단어 생성에는 부족 파인튜닝 소량의 데이터로도 특정 작업에 빠르게 적응

BERT의 구조

BERT는 다음과 같이 구성됩니다:

  • Token Embedding: 단어를 벡터로 변환
  • Position Embedding: 단어의 위치 정보 추가
  • Segment Embedding: 문장 구분 정보 추가 ([CLS], [SEP] 토큰)
  • 12-24개의 Transformer Encoder 층

Fine-tuning: 전이학습

BERT의 강력한 특징은 Fine-tuning입니다:

  • 큰 데이터로 사전학습된 일반적 지식 활용
  • 작은 데이터로 특정 작업에 빠르게 적응
  • 과적합 위험 감소
  • 학습 시간 단축

BERT 변형 모델 비교

BERT 출시 이후 등장한 주요 개선 모델들:

모델 제안 핵심 개선 성능 RoBERTa Meta (2019) 더 많은 데이터, 더 많은 학습 단계, NSP 제거 BERT 대비 약간 향상 ALBERT Google (2019) 파라미터 공유, 차원 감소로 모델 크기 70% 축소 같은 성능, 훨씬 빠름 DistilBERT Hugging Face (2019) 지식 증류로 모델 크기 40% 축소 97% 성능, 60% 빠름 ELECTRA Google (2020) 판별자 사전학습 방식 변경 (MLM 대신 GAN) 작은 데이터에서 우수한 성능 mBERT Google (기본) 104개 언어 동시 학습 다국어 지원 XLM-RoBERTa Meta (2019) 100개 언어 지원, RoBERTa 구조 다국어 최고 성능
응용 분야: 감정분석, 개체명인식(NER), 질의응답(QA), 텍스트 분류, 텍스트 유사도 계산 등

6. GPT 계열

GPT의 기본 원리

GPT (Generative Pre-trained Transformer)는 Transformer의 Decoder만 사용합니다. 자기회귀(Autoregressive) 방식으로 다음 단어를 계속 예측합니다.

GPT의 학습 방식: "이전 단어들이 주어졌을 때 다음 단어는 무엇인가?" 를 계속 반복하여 문장을 생성하도록 학습합니다.

GPT의 발전 과정: 상세 비교

버전별 상세 설명

GPT-1 (2018년) - 기초

  • 파라미터: 1억 1,700만 개
  • 특징: Transformer Decoder 구조 최초 적용
  • 성능: 문장 분류, 텍스트 유사도 측정 가능
  • 의의: 사전학습의 가능성 입증

GPT-2 (2019년) - 확장

  • 파라미터: 15억 개 (GPT-1의 10배 이상)
  • 특징: Zero-shot Learning 가능 (특정 작업으로 학습하지 않음)
  • 성능: 사전학습만으로도 다양한 텍스트 생성 가능
  • 의의: "스케일이 곧 능력"임을 증명

GPT-3 (2020년) - 혁신

  • 파라미터: 1,750억 개 (어마어마한 규모)
  • 특징: In-context Learning 가능 - 프롬프트만으로 새로운 작업 학습
  • Few-shot: 극히 적은 예시(보통 2-5개)로도 새로운 작업 학습
  • 응용: 기계번역, 질의응답, 코드 생성, 수학 문제 풀이
  • 한계: 아직 Fine-tuning이 필요한 경우도 있음

GPT-3.5 (2022년) - ChatGPT

  • 개선: 대화형 UI 추가로 사용성 극대화
  • 특징: RLHF(강화학습)로 인간 의도에 맞게 조정
  • 성능: GPT-3 대비 훨씬 높은 품질의 답변, 안전성 향상
  • 의의: AI가 대중에 널리 알려지는 계기 (월 1억 사용자)

GPT-4 (2023년) - 최신

  • 파라미터: 정확한 수치 미공개 (1조 개 이상 추정)
  • 멀티모달: 이미지와 텍스트 동시 처리 가능
  • 성능: 인간 수준의 이해와 추론 능력
  • 특징: 복잡한 문제 해결, 장문 문맥 처리(최대 128K 토큰)
  • 신뢰도: 더욱 신뢰할 수 있는 답변 생성

GPT의 핵심 특징

특징 설명 자기회귀성 이전 출력이 다음 입력이 됨. 단어를 하나씩 생성 단방향성 좌측에서 우측으로만 문맥을 봄 텍스트 생성 Decoder만 있으므로 자연스러운 문장 생성 가능 대화형 프롬프트에 따라 유연한 응답 생성 스케일 효과 모델 크기와 데이터량이 커질수록 성능 향상
발전 과정: GPT-1은 기본 문법을 아는 학생, GPT-2는 다양한 글쓰기를 경험한 고등학생, GPT-3은 많은 책을 읽은 지식인, GPT-4는 전문가 수준이라고 할 수 있습니다.
현재 트렌드: 점점 더 큰 모델을 만들되, 효율성 개선과 멀티모달 확장이 진행 중입니다.

7. BERT vs GPT 비교

구조 비교

항목 BERT GPT 기반 구조 Transformer Encoder Transformer Decoder 방향성 양방향 (Bidirectional) 단방향 (Left-to-Right) 문맥 이해 앞뒤 모두 봄 앞에서만 봄 Attention 제약 없음 (모든 위치 접근) Causal Mask (미래 차단)

학습 방식 비교

항목 BERT GPT 학습 목표 MLM (빈칸 채우기) LM (다음 단어 예측) 훈련 방식 양쪽 방향 동시 학습 순차적 예측 학습 추가 목표 NSP (문장 관계 예측) 없음 사전학습 데이터 Wikipedia, BookCorpus 인터넷 텍스트 (대규모)

강점과 약점

항목 BERT GPT 텍스트 이해 탁월 - 양방향 문맥 제한적 - 단방향 텍스트 생성 부족 - Encoder만 있음 탁월 - Decoder 최적화 대화 능력 약함 강함 감정분석 우수 보통 질의응답 우수 우수 개체명인식 우수 보통

응용 사례 비교

BERT 활용 GPT 활용 감정분석 (영화평 분류) 챗봇 (ChatGPT) 개체명인식 (인물 추출) 코드 생성 질의응답 시스템 창의적 글쓰기 텍스트 분류 번역 검색 시스템 (임베딩) 요약

선택 기준

BERT를 선택해야 할 때:
  • 텍스트 의미 이해가 중심인 작업
  • 분류, 감정분석, NER 등의 이해 기반 작업
  • 상대적으로 작은 모델 크기 필요
  • 미세조정(Fine-tuning)으로 빠른 구현 필요
GPT를 선택해야 할 때:
  • 텍스트 생성이 중심인 작업
  • 대화, 챗봇, 질의응답 등의 생성 기반 작업
  • 창의적인 글쓰기 필요
  • Zero-shot / Few-shot 학습 필요
최신 트렌드: BERT와 GPT의 장점을 모두 가진 모델들이 등장했습니다. 예: T5 (Encoder-Decoder), ELECTRA, 그리고 최신 GPT-4는 이해와 생성 모두 탁월합니다.

8. 멀티모달 AI

멀티모달의 개념

지금까지의 모델은 주로 텍스트만 처리했습니다. 멀티모달 AI는 여러 종류의 데이터를 동시에 이해하고 처리합니다.

멀티모달 AI: 텍스트, 이미지, 음성, 비디오 등 다양한 형태의 데이터를 함께 처리하고 이해하는 AI 시스템.

멀티모달 시스템의 구성

1. 입력 모듈: 다양한 데이터 처리

각 데이터 종류를 전문 신경망으로 처리합니다:

  • 텍스트: NLP 모델 (BERT, GPT 등) - 문자 시퀀스 처리
  • 이미지: CNN (합성곱 신경망) - 시각 특징 추출
  • 음성: 음성 인코더 - 스펙트로그램 변환 및 처리
  • 비디오: 3D CNN - 시간 축 정보 포함

2. 융합 모듈 (Fusion): 정보 통합 상세

각각 처리된 정보를 하나의 의미로 종합하는 세 가지 전략:

융합 방식 개념 적용 시점 장점 단점 Early Fusion 입력 초기 단계에서 모달 결합 인코더 이전 강한 상호작용, 모달 간 정보 교환 다른 모달별 처리 불가, 대역폭 증가 Late Fusion 각 모달을 독립 처리 후 끝에서 합침 분류층 이전 모달별 전문 처리 가능, 간단함 약한 상호작용, 정보 손실 Hybrid Fusion 초기와 후기 융합 모두 사용 중간/끝단계 강한 상호작용 + 전문 처리 복잡도 증가, 계산 비용 증가

Cross-Attention (교차 주의)

텍스트와 이미지 간의 깊은 상호작용:

  • 개념: 한 모달의 Query가 다른 모달의 Key-Value에 주의
  • 예시: 텍스트 "빨간 자동차"의 Query가 이미지의 어느 영역(색깔, 형태)을 봐야 하는지 학습
  • 효과: 텍스트와 이미지의 의미적 정렬 달성

대조학습 (Contrastive Learning)

  • 원리: "같은 의미 쌍은 임베딩 공간에서 가깝게, 다른 의미는 멀게"
  • 예시: 이미지와 그 설명이 비슷한 방향의 벡터, 무관한 텍스트는 먼 벡터
  • 활용: CLIP 모델이 대표적 - 이미지-텍스트 검색에 최적화
유추: 인간이 뉴스를 이해할 때 기자의 말(음성), 자막(텍스트), 화면 영상(이미지)을 모두 종합하는 것과 같습니다.

3. 지식 증류 (Knowledge Distillation)

큰 모델의 지식을 작은 모델로 옮깁니다:

  • Teacher 모델: 큰 멀티모달 모델 (높은 성능)
  • Student 모델: 작은 모델 (빠른 추론)
  • Student가 Teacher의 확률 분포를 모방하도록 학습
  • 결과: 작은 모델도 우수한 성능 달성

4. 출력 모듈: 생성 및 응답

  • Vision-Language Model: 이미지 설명 생성 또는 텍스트에 맞는 이미지 생성
  • Diffusion Model: 텍스트 설명으로부터 새로운 이미지 합성
  • Speech Synthesis: 텍스트를 음성으로 변환

Cross-Attention: 모달 간 상호작용

멀티모달 모델의 핵심 메커니즘:

  • 텍스트의 Query가 이미지 영역의 Key-Value에 주의
  • 각 단어가 이미지의 어느 부분과 연관된지 학습
  • 이미지도 텍스트의 어느 부분과 연관된지 학습
  • 결과: 깊은 의미적 융합

Vision Language Adapter (VLA)

시각-언어 모델 간의 효율적인 통합:

  • 이미지 인코더와 LLM을 연결하는 어댑터
  • 이미지 토큰을 LLM이 이해할 수 있는 형식으로 변환
  • 최소한의 추가 학습으로 통합 가능
미래: 향후 음성, 영상, 센서 데이터, 실시간 입력까지 포함한 더욱 복잡한 멀티모달 AI가 등장할 것으로 예상됩니다. 이는 인간의 감각 능력에 더 가까워질 것입니다.

9. LLM 훈련 과정

LLM 훈련의 세 단계

현대 대형 언어 모델은 다음 세 단계를 거쳐 훈련됩니다:

Pre-training
→
SFT
→
RLHF

1단계: Pre-training (사전학습)

기초적인 언어 능력 획득:

  • 목표: 다음 토큰 예측 (Next Token Prediction)
  • 데이터: 인터넷의 거대한 텍스트 데이터 (수조 토큰)
  • 방식: 자기지도학습 (Self-Supervised Learning)
  • 결과: 기본적인 언어 패턴과 일반 지식 학습

예시:

  • "인공지능은 → 컴퓨터 시스템으로..."
  • "사과는 → 빨간색 과일로..."
  • 수십억 번의 반복
특징: 이 단계에서 모델은 사실적 오류가 많을 수 있으며, 유해한 내용도 학습할 수 있습니다.

2단계: SFT (Supervised Fine-Tuning)

특정 작업에 맞게 조정:

  • 목표: 사전학습 모델을 특정 형식의 대화나 작업에 맞추기
  • 데이터: 고품질의 레이블된 데이터 (보통 수만~수십만)
  • 방식: 지도학습 (Supervised Learning)
  • 예시: "Q: 날씨는? A: 저는 실시간 날씨 정보가 없습니다"

SFT의 특징:

  • 인간 전문가가 고품질 답변 작성
  • 대화형 형식으로 모델을 훈련
  • 모델이 "맞는 답변 형식"을 학습
  • 일반화 능력은 pre-training에서 나옴

3단계: RLHF (Reinforcement Learning from Human Feedback)

인간의 선호도에 따라 조정:

  • 목표: 모델의 답변이 인간의 가치관과 선호도에 부합하도록 조정
  • 방식: 강화학습 (Reinforcement Learning)
  • 보상 모델: 인간의 평가를 학습한 별도의 모델

RLHF의 구체적 과정

1단계: 보상 모델 훈련
- SFT 모델이 생성한 여러 답변을 인간이 평가
- "이 답변이 더 좋다/나쁘다" 라는 비교 데이터 수집
- 이 평가를 학습한 보상 모델(Reward Model) 생성

2단계: 정책 최적화
- SFT 모델이 다양한 답변 생성
- 보상 모델이 각 답변에 점수 부여
- 점수를 최대화하도록 모델의 가중치 조정
- PPO(Proximal Policy Optimization) 같은 강화학습 알고리즘 사용

RLHF의 장점

  • 모델의 답변이 더 안전하고 정직함
  • 사용자의 선호도를 반영
  • 유해한 콘텐츠 생성 감소
  • 설명가능성(Explainability) 향상

ChatGPT의 예시

ChatGPT의 훈련 과정:

  • Pre-training: GPT-3 사전학습
  • SFT: 인간이 작성한 고품질 대화 데이터로 미세조정
  • RLHF: 인간 피드백을 기반으로 강화학습 (InstructGPT → ChatGPT)
비용: RLHF 단계는 매우 비용이 많이 들어갑니다. 수천~수만 명의 인간이 답변을 평가해야 합니다.

최신 트렌드

  • DPO (Direct Preference Optimization): RLHF를 더 효율적으로
  • Constitutional AI: 인간 피드백 대신 AI 헌법을 기준으로
  • Mixture of Experts (MoE): 효율성 향상
  • 양자화: 모델 크기 감소

10. 토큰화와 임베딩

토큰화 (Tokenization)

신경망은 텍스트를 직접 처리할 수 없습니다. 먼저 텍스트를 작은 단위로 분할합니다.

토큰화: 텍스트를 신경망이 처리할 수 있는 작은 단위(토큰)로 분할하는 과정.

토큰화의 종류

1. 단어 수준 토큰화 (Word-level)

  • 방식: 공백과 구두점 기준으로 분할
  • 예: "Hello world" → ["Hello", "world"]
  • 장점: 간단하고 직관적
  • 단점: 어휘 크기가 너무 커서 메모리 낭비, 희귀 단어 처리 어려움

2. 문자 수준 토큰화 (Character-level)

  • 방식: 문자 하나하나로 분할
  • 예: "hello" → ["h", "e", "l", "l", "o"]
  • 장점: 미지의 단어도 처리 가능
  • 단점: 시퀀스가 너무 길어서 느림

3. 서브워드 토큰화 (Subword-level) - 현재 표준

현대 NLP의 표준. 단어와 문자의 장점을 결합합니다:

방식 특징 사용 모델 BPE
(Byte Pair Encoding) 가장 자주 나오는 연속 토큰을 병합하는 방식. 반복적으로 병합하며 어휘 구성 GPT, RoBERTa WordPiece BPE와 유사하지만, 병합 시 우도(Likelihood)를 기준으로 선택 BERT SentencePiece 언어 무관적. 문자 또는 음소 기반으로 시작하여 병합. 다국어 처리 최적화 T5, mBERT, XLNet

BPE (Byte Pair Encoding) 상세 알고리즘

BPE의 작동 원리를 단계별로 시각화:

0
초기 상태
각 문자를 개별 토큰으로 분할
"apple" → ['a', 'p', 'p', 'l', 'e']
vocabulary = {a, p, l, e}
1
가장 빈번한 쌍 찾기
연속된 토큰 쌍 중 가장 많이 나타나는 것 선택
('p', 'p')가 가장 빈번
빈도 = 1회
2
병합
"pp" → 새로운 토큰 'pp'
"apple" → ['a', 'pp', 'l', 'e']
vocabulary = {a, pp, l, e}
3
반복
다음 빈번한 쌍 찾기
('p', 'l') 또는 ('l', 'e')...
계속 반복 (보통 수십만 번)
4
최종 어휘 구성
합의된 어휘 크기에 도달할 때까지 반복
예: 50,000개 어휘
"apple" → ['a', 'pp', 'l', 'e'] 또는 다른 분할

BPE의 특징

  • 학습 단계: 학습 데이터에서 가장 빈번한 쌍들부터 병합
  • 결정론적: 같은 데이터에서는 항상 같은 결과
  • 미지의 단어: 학습되지 않은 단어도 문자로 분할하여 표현 가능
  • 언어 무관: 모든 언어에 적용 가능
  • 효율성: 단어 전체를 어휘에 저장할 필요 없음

WordPiece와 SentencePiece 비교

BPE와 비교하여 개선된 토큰화 방식:

방식 특징 우도 기반 사용 모델 BPE 가장 빈번한 쌍을 병합 아니오 - 빈도 기반 GPT, RoBERTa WordPiece 병합 시 우도(확률) 최대화 기준
(새로운 토큰이 얼마나 "유용한가") 예 - 우도 기반 BERT, DistilBERT SentencePiece 언어 무관적. 문자 또는 음소에서 시작하여 BPE 적용.
공백도 특수 토큰으로 처리 예 - 우도 기반 T5, mBERT, XLNet

임베딩 (Embedding)

토큰화 후에는 각 토큰을 벡터로 변환해야 합니다. 이것이 임베딩입니다.

임베딩: 단어나 토큰을 연속된 수의 벡터로 표현하는 방법. 의미적으로 유사한 단어는 벡터 공간에서도 가깝게 위치합니다.

임베딩의 종류

1. Word2Vec - 정적 임베딩

2013년 Google에서 제안. 각 단어에 고정된 벡터를 부여합니다.

Word2Vec의 두 가지 구조
방식 원리 예시 특징 CBOW
(Continuous
Bag of Words) 주변 단어들(context)로부터
중심 단어를 예측 "dog good is ___" → "boy"
입력: [dog, good, is]
출력: boy 문맥 윈도우 크기 지정 필요
빠르고 작은 데이터에 강함 Skip-gram 중심 단어로부터
주변 단어들을 예측 "boy" → "good", "dog", "is"
입력: boy
출력: [good, dog, is] 드문 단어 표현 더 좋음
큰 코퍼스에서 성능 우수
Word2Vec의 구조 상세

Skip-gram의 신경망 구조:

입력층: 중심 단어의 One-hot 벡터
은닉층: d차원의 dense layer (보통 d=300)
→ 이 층의 가중치가 Word2Vec 임베딩!
출력층: Context 단어들의 확률 분포

학습: 가중치를 조정하여 중심 단어로부터 주변 단어를 정확히 예측하도록
Word2Vec의 특징
  • 장점: 빠르고 효율적, 벡터 연산이 가능 ("king" - "man" + "woman" ≈ "queen")
  • 단점: 같은 단어는 항상 같은 벡터 (다의성 미처리)
  • 예: "bank"가 금융기관일 때와 강(river bank)일 때 같은 벡터
  • 개선: Negative Sampling, Hierarchical Softmax로 학습 가속화

2. GloVe - 전역 벡터 (Global Vectors)

2014년 Stanford에서 제안. Word2Vec과 행렬 분해를 결합합니다.

  • 개념: 전체 말뭉치의 단어 공출현(Co-occurrence) 행렬을 활용
  • 동작: Co-occurrence 행렬을 저차원으로 분해하여 임베딩 추출
  • 장점: 전역적 정보(전체 통계)와 지역적 정보(인접 단어)를 동시에 활용
  • 성능: Word2Vec보다 의미 파악에 더 우수 (많은 경우)
  • 활용: NLP 다양한 작업에서 기본 임베딩으로 사용
GloVe의 수학적 기초
X: 단어 공출현 행렬 (i번째 단어가 j번째 단어와 함께 나타난 횟수)
목표: X ≈ W_i · W_j^T (행렬 인수분해)
W_i: 단어 i의 임베딩
W_j: 단어 j의 컨텍스트 임베딩

손실함수: L = Σ f(X_ij) * (W_i · W_j^T - log(X_ij))²
f(X_ij): 빈도 가중 함수 (희귀 쌍에 가중치 낮춤)

3. FastText

Facebook에서 제안. Word2Vec의 개선판:

  • 문자 수준 n-gram 정보 사용
  • 미지의 단어도 표현 가능 (철자 유사성 활용)
  • 형태적으로 유사한 단어들 잘 처리

정적 vs 동적 임베딩 비교
정적 임베딩 (Word2Vec, GloVe):
- 각 단어: 1개의 고정된 벡터
- 다의성 처리 불가 ("bank" = 금융 기관 = 강의 둑)
- 빠르고 메모리 효율적

동적 임베딩 (ELMo, BERT, GPT):
- 각 단어: 문맥에 따라 여러 벡터
- 다의성 자동 처리 (같은 단어도 다른 벡터)
- 더 정확하지만 계산 비용 높음

결론: 현대 NLP는 동적 임베딩(BERT, GPT)이 표준

임베딩의 성질

좋은 임베딩의 특징:

  • 의미 유사성: 의미가 비슷한 단어는 벡터도 가까움
  • 벡터 연산: "king" - "man" + "woman" ≈ "queen"
  • 선형 성질: 의미 관계가 벡터 공간에서 선형으로 표현
  • 저차원성: 보통 300차원 정도로 충분 (Word2Vec 기준)
최신 트렌드: Contextual Embedding (BERT, GPT)이 정적 임베딩(Word2Vec, GloVe)을 대체하고 있습니다. 같은 단어도 문맥에 따라 다르게 표현되므로 성능이 훨씬 좋습니다.

임베딩의 응용

  • 유사도 검색: 코사인 유사도로 관련 문서 검색
  • 군집화: K-means로 단어 그룹화
  • 분류: 임베딩을 특징으로 사용해 분류 모델 학습
  • 추천 시스템: 사용자-아이템 임베딩 공간에서 추천

핵심 요약

  • Seq2Seq: 입력을 압축하여 출력 생성하는 구조이지만, 고정 벡터로 인한 정보 손실 문제 존재
  • Attention: 출력 시 입력의 관련 부분에만 집중하여 정보 손실 극복. Query-Key-Value 메커니즘
  • Self-Attention: 문장 내 단어들 간의 관계를 파악하는 메커니즘
  • Multi-Head Attention: 여러 관점에서 동시에 주의를 기울여 다양한 관계 포착
  • Positional Encoding: 사인/코사인 함수로 단어의 위치 정보 인코딩
  • Transformer: Attention만으로 구성되어 병렬 처리 가능한 구조. 현대 AI의 기초 아키텍처
  • BERT: 양방향 Transformer Encoder. 텍스트 이해에 탁월. MLM과 NSP로 학습
  • GPT: 단방향 Transformer Decoder. 텍스트 생성에 탁월. 다음 토큰 예측으로 학습
  • BERT vs GPT: BERT는 이해 중심, GPT는 생성 중심. 구조와 학습 방식이 다름
  • 멀티모달 AI: 텍스트, 이미지, 음성 등을 동시에 처리하는 시스템
  • LLM 훈련: Pre-training → SFT → RLHF 세 단계로 진행. 각 단계가 다른 역할
  • 토큰화: BPE, WordPiece, SentencePiece 등의 서브워드 방식이 표준
  • 임베딩: Word2Vec, GloVe (정적)에서 BERT, GPT (동적)로 진화. 동적이 더 우수
