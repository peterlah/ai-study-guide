1. AI 에이전트 개요

AI 에이전트의 정의

AI 에이전트는 주어진 환경을 인식하고, 목표를 향해 자율적으로 의사결정을 내리며, 행동을 실행하는 지능형 소프트웨어 또는 로봇 시스템입니다.

AI 에이전트 (AI Agent) - 센서를 통해 환경을 감지하고, 내부 의사결정 로직에 따라 행동을 선택하며, 액튜에이터를 통해 환경에 영향을 미치는 자율 시스템. 목표 달성을 위해 학습하고 적응합니다.

에이전트 유형 분류

1) 반사 에이전트 (Reflex Agent)

입력에 대해 직접적으로 반응하는 가장 단순한 형태입니다. 규칙 기반 시스템으로, 조건-행동 쌍으로 정의됩니다.

예시: 손을 뜨거운 물에 넣으면 자동으로 빼는 반사 행동처럼, 입력이 들어오면 즉시 정해진 행동을 수행합니다.

2) 모델 기반 에이전트 (Model-based Agent)

세상의 상태를 모델링하여 미래를 예측하고 행동합니다. 과거의 경험과 환경 모델을 유지합니다.

3) 목표 기반 에이전트 (Goal-based Agent)

명확한 목표를 설정하고, 그 목표를 달성하기 위한 최적의 행동 순서를 계획합니다.

4) 효용 기반 에이전트 (Utility-based Agent)

행동의 "효용" 또는 선호도를 정량화하여, 가장 좋은 결과를 낼 수 있는 행동을 선택합니다.

5) 학습하는 에이전트 (Learning Agent)

경험으로부터 학습하여 점진적으로 성능을 개선합니다. 강화학습(Reinforcement Learning)이 대표적입니다.

에이전트의 핵심 특성

  • 자율성: 외부 제어 없이 독립적으로 행동
  • 반응성: 환경 변화에 신속하게 대응
  • 주도성: 목표 달성을 위해 능동적으로 행동
  • 사회성: 다른 에이전트나 인간과 협력
시험 포인트: AI 에이전트는 단순 반사에서 시작하여 학습 능력까지 발전하는 계층적 구조를 가집니다.

2. 에이전틱 AI (Agentic AI)

에이전틱 AI의 정의

에이전틱 AI는 대형 언어 모델을 기반으로, 자율적으로 복잡한 다단계 작업을 계획하고 실행하는 고급 AI 시스템입니다. 목표 설정, 자체 계획, 실행, 피드백 수집의 자율 루프를 반복합니다.

에이전틱 AI - 대규모 언어 모델을 활용하여 자율적으로 복잡한 작업을 수행하는 AI. 목표 설정부터 실행 및 피드백까지의 완전한 사이클을 자동화합니다.

기존 AI vs 에이전틱 AI

특성 기존 AI (요청-응답) 에이전틱 AI (자율 루프)
작동 방식 사용자 입력 -> AI 응답 목표 설정 -> 계획 -> 실행 -> 피드백
자율성 낮음 (지시 필요) 높음 (자체 판단)
복잡도 단일 작업 다단계 작업
의사결정 규칙 기반 동적 적응
오류 처리 외부 개입 필요 자체 수정
활용 사례 챗봇, Q&A 소프트웨어 개발, 데이터 분석, 프로젝트 관리

에이전틱 AI의 자율 루프 (Autonomous Loop)

1. 목표 설정 → 2. 계획 수립 → 3. 실행 → 4. 피드백 → (반복)

1단계: 목표 설정 (Goal Setting)

달성해야 할 최종 목표와 부분 목표들을 설정합니다. 예: "6개월 내 매출 20% 증대" -> "고객 확보", "구매 전환율 개선" 등

2단계: 계획 수립 (Planning)

목표를 달성하기 위한 구체적인 행동 계획을 수립합니다. 필요한 도구, 리소스, 일정을 결정합니다.

3단계: 실행 (Execution)

계획된 행동을 실제로 수행합니다. 코드 작성, 데이터 분석, API 호출 등을 자동으로 실행합니다.

4단계: 피드백 (Feedback)

실행 결과를 평가하고, 목표 달성 정도를 측정합니다. 필요시 계획을 수정하고 다시 실행합니다.

유사 개념: 에이전틱 AI는 마치 능력 있는 프로젝트 매니저처럼, 전체 프로젝트를 자율적으로 계획하고 실행하며, 진행 상황을 모니터링합니다.

에이전틱 AI의 핵심 능력

1) 도구 사용 (Tool Use)

필요한 도구를 식별하고 적절히 사용합니다. (웹 검색, 데이터베이스 쿼리, API 호출, 파일 작성 등)

2) 계획 수정 (Plan Adjustment)

실행 중 문제가 발생하면 계획을 즉시 수정합니다.

3) 오류 분석 및 복구 (Error Recovery)

오류가 발생했을 때 원인을 분석하고 대체 방안을 실행합니다.

4) 멀티스텝 추론 (Multi-step Reasoning)

복잡한 문제를 여러 단계로 나누어 논리적으로 해결합니다.

에이전틱 AI의 활용 사례

• 소프트웨어 개발: 자동 코드 생성 및 테스트

• 데이터 분석: 데이터 수집 -> 정제 -> 분석 -> 보고서 작성

• 고객 서비스: 복잡한 문제 자율 해결

• 재정 관리: 예산 계획 및 최적화

• 영업: 리드 발굴, 제안서 작성, 계약 관리

에이전틱 AI의 미래 영향

  • 업무 자동화 수준의 획기적 향상
  • 인간-AI 협업의 새로운 형태
  • 조직 구조와 직무의 변화
  • 새로운 가치 창출 기회
시험 포인트: 에이전틱 AI는 "자율 루프"를 핵심으로 합니다. 목표 설정부터 피드백까지 인간 개입 없이 자동 진행됩니다.

3. AI 가상 비서 (AI Virtual Assistant)

AI 가상 비서의 정의

AI 가상 비서는 음성이나 텍스트를 통해 사용자와 상호작용하며, 정보 제공, 일정 관리, 쇼핑, 엔터테인먼트 등 다양한 서비스를 제공하는 지능형 보조 시스템입니다.

AI 가상 비서 - 사용자의 자연어 명령을 이해하고, 필요한 정보를 검색하거나 작업을 실행하며, 자연스러운 음성 또는 텍스트로 응답하는 AI 시스템.

주요 기능

1) 정보 검색 및 제공

날씨, 뉴스, 주식 정보, 일반적인 질문에 대한 답변 등을 신속하게 제공합니다.

2) 일정 및 작업 관리

캘린더에 일정 추가, 리마인더 설정, 할 일 관리를 음성 명령으로 처리합니다.

3) 스마트 홈 제어

조명, 온도, 보안 시스템 등 IoT 기기를 음성으로 제어합니다.

4) 쇼핑 및 결제

"Amazon에서 책을 주문해 줄래?" 같은 음성 명령으로 구매를 진행합니다.

5) 엔터테인먼트

음악 재생, 팟캐스트 스트리밍, 오디오북 청취 등을 제어합니다.

6) 개인화 학습

사용자의 선호도와 습관을 학습하여 점진적으로 더 나은 서비스를 제공합니다.

AI 가상 비서의 활용 사례

개인용

• 아침에 깨어나면 날씨와 일정 자동 읽어주기

• 출근 중 뉴스와 이메일 요약

• 음성으로 쇼핑 리스트 추가

기업용

• 직원의 회의 예약 자동화

• 고객 상담 첫 단계 처리

• 내부 시스템 접근 및 정보 조회

헬스케어

• 약 복용 리마인더

• 기본적인 건강 상담

• 의사 예약 자동 관리

시험 포인트: AI 가상 비서는 "음성 인식 + 자연어 처리 + 작업 실행"의 통합 시스템입니다. 사용자 맞춤화와 프라이버시 보호가 중요합니다.

4. MCP (Model Context Protocol)

MCP의 정의

Model Context Protocol은 AI 모델과 외부 도구, 데이터 소스 간의 표준화된 통신을 위한 프로토콜입니다. AI 에이전트가 다양한 외부 시스템과 안전하게 상호작용할 수 있게 합니다.

MCP (Model Context Protocol) - AI 모델이 외부 도구, API, 데이터베이스에 접근하기 위한 표준 인터페이스. Host, Client, Server의 3계층으로 구성된 개방형 프로토콜.

MCP의 구조: Host / Client / Server

1) Host (호스트) - AI 모델 또는 애플리케이션

사용자의 요청을 받아 처리하고, 필요한 경우 외부 도구를 호출합니다. 예: ChatGPT, Claude, LLM 애플리케이션

2) Client (클라이언트) - 호스트 내 라우터

호스트와 서버 간의 통신을 중개합니다. 호스트의 요청을 서버에 전달하고, 응답을 호스트에 반환합니다.

3) Server (서버) - 외부 도구 또는 데이터 소스

실제 기능을 수행하는 외부 시스템입니다. 예: 웹 검색 API, 데이터베이스, 파일 시스템, 계산 엔진

Host (AI) → Client (라우터) → Server (도구)

MCP의 보안 취약점

1) 권한 상승 (Privilege Escalation)

AI 모델이 예상보다 높은 권한으로 외부 시스템에 접근할 위험이 있습니다.

위험: "파일 삭제" 권한만 필요한데 "전체 시스템 제어" 권한을 가질 수 있음

2) 정보 유출 (Information Leakage)

외부 서버와의 통신 과정에서 민감한 정보가 노출될 수 있습니다.

3) 인증 우회 (Authentication Bypass)

MCP 프로토콜의 인증 메커니즘이 약할 경우, 무단 접근이 가능합니다.

4) 명령어 인젝션 (Command Injection)

사용자 입력이 제대로 검증되지 않으면, 악의적인 명령어가 주입될 수 있습니다.

5) 중간자 공격 (Man-in-the-Middle, MITM)

클라이언트와 서버 간의 통신이 암호화되지 않으면, 통신 내용을 도청할 수 있습니다.

MCP 보안 취약점 대응방안

취약점 대응방안
권한 상승 최소 권한 원칙 (Least Privilege) 적용, 역할 기반 접근 제어 (RBAC)
정보 유출 암호화된 통신 (TLS/SSL), 민감한 정보 마스킹, 접근 로그 관리
인증 우회 강력한 인증 메커니즘 (OAuth 2.0, JWT), 멀티팩터 인증
명령어 인젝션 입력 검증 및 정제, 매개변수화된 쿼리, 화이트리스트 방식
중간자 공격 TLS/SSL 암호화, 인증서 검증, 핀 고정 (Certificate Pinning)
시험 포인트: MCP는 AI와 외부 도구의 다리 역할을 하지만, 보안 취약점이 많습니다. "최소 권한 원칙"과 "암호화 통신"이 핵심입니다.

5. Agent-to-Agent 통신 (A2A)

A2A의 정의

Agent-to-Agent (A2A) 통신은 여러 AI 에이전트들이 서로 협력하여 복잡한 작업을 수행하는 메커니즘입니다. 에이전트 간 메시지 전달, 정보 공유, 협력을 가능하게 합니다.

A2A (Agent-to-Agent) - 독립적인 AI 에이전트들이 표준화된 인터페이스를 통해 상호 통신하고 협력하는 체계. RESTful API, 메시지 브로커, 공유 메모리 등을 활용합니다.

A2A의 핵심 개념: RESTful 에이전트 통신

1) RESTful 아키텍처

각 에이전트가 HTTP/REST API를 제공하여, 다른 에이전트들이 이 API를 호출하여 기능을 요청합니다.

예시: 영업 에이전트가 "재고 확인 에이전트"의 REST API를 호출하여 재고를 조회하고, 결과를 고객에게 전달합니다.

2) 메시지 브로커

에이전트들이 직접 통신하지 않고, 중앙의 메시지 브로커를 통해 비동기로 메시지를 주고받습니다. (예: RabbitMQ, Kafka)

3) 공유 메모리 (Shared Context)

모든 에이전트가 접근할 수 있는 공유 메모리 또는 데이터베이스에 정보를 저장하여, 상태를 동기화합니다.

MCP vs A2A vs ACP vs ANP 비교

프로토콜 목적 통신 방식 사용 대상
MCP (Model Context Protocol) AI 모델과 외부 도구 연결 표준화된 인터페이스 AI 모델 ↔ 외부 시스템
A2A (Agent-to-Agent) 에이전트 간 협력 RESTful API, 메시지 큐 에이전트 ↔ 에이전트
ACP (Agent Communication Protocol) 멀티 에이전트 시스템 통신 FIPA 표준 프로토콜 복수 에이전트 간 표준화
ANP (Agent Network Protocol) 에이전트 네트워크 구성 P2P 또는 중앙식 아키텍처 대규모 에이전트 네트워크

A2A 활용 사례

예시 1: 이커머스 주문 처리

고객 서비스 에이전트 -> 재고 확인 에이전트 -> 결제 에이전트 -> 배송 에이전트

예시 2: 데이터 분석 파이프라인

데이터 수집 에이전트 -> 정제 에이전트 -> 분석 에이전트 -> 보고서 생성 에이전트

예시 3: 자율주행 차량 플릿

각 차량(에이전트) ↔ 교통 관리 에이전트 ↔ 긴급 상황 대응 에이전트

시험 포인트: A2A는 "에이전트 간 협력"을 가능하게 하는 핵심 기술입니다. MCP는 AI-도구, A2A는 에이전트-에이전트 통신이라고 구분합니다.

6. 물리 AI (Physical AI)

물리 AI의 정의

물리 AI는 디지털 공간이 아닌 현실 세계에서 작동하는 AI 시스템입니다. 센서를 통해 환경을 인식하고, 액튜에이터를 통해 물리적 행동을 수행합니다.

물리 AI (Physical AI) - 로봇, 자율주행차 등 현실 세계에 물리적 신체를 가지고 작동하는 AI. 센싱, 지각, 의사결정, 행동의 완전한 사이클을 수행합니다.

임보디드 AI (Embodied AI)

임보디드 AI는 물리적 신체(body)를 가진 AI를 의미합니다. 단순한 계산이 아니라 실제 환경과 상호작용합니다.

차이점: 소프트웨어 AI는 머릿속으로만 생각하는 철학자, 임보디드 AI는 손과 발로 행동하는 장인입니다.

물리 AI의 핵심 기술 요소

1) 센싱 (Sensing)

환경을 인식하기 위한 센서들

  • 카메라: 시각 정보
  • 라이더(LiDAR): 3D 거리 정보
  • 레이더: 움직이는 객체 감지
  • 초음파: 장애물 감지
  • IMU: 움직임 및 자세 정보

2) 컴퓨터 비전 (Computer Vision)

센서로부터 받은 영상 정보를 처리하여 의미 있는 정보를 추출합니다.

3) 멀티모달 처리 (Multimodal Processing)

여러 센서 정보(시각, 음성, 촉각)를 통합하여 환경을 이해합니다.

4) 강화학습 (Reinforcement Learning, RL)

시행착오를 통해 최적의 행동 정책을 학습합니다.

5) 엣지 AI (Edge AI)

클라우드에 의존하지 않고, 로컬 장치에서 AI 모델을 실행합니다. 저지연(Low Latency) 반응이 필요합니다.

6) Sim-to-Real (시뮬레이션에서 현실로)

시뮬레이션 환경에서 학습한 AI 모델을 현실 세계의 로봇에 적용합니다.

물리 AI의 주요 응용 분야

자율주행 (Autonomous Vehicles)

센서 데이터 -> 환경 인식 -> 의사결정 -> 조향/가속/제동

로봇공학 (Robotics)

제조, 서비스, 배송, 의료 등 다양한 분야의 로봇

드론 (Drones)

자율 비행, 물체 감지 및 추적

의료 로봇 (Medical Robots)

수술 로봇, 재활 로봇, 돌봄 로봇

시험 포인트: 물리 AI는 "현실 세계에서의 자율 행동"이 핵심입니다. 센싱부터 액션까지의 완전한 루프가 필요합니다.

8. 휴머노이드 로봇

휴머노이드 로봇의 정의

휴머노이드 로봇은 인간의 형태(머리, 팔, 다리, 몸통)와 행동 방식을 닮은 로봇입니다. 인간 중심으로 설계된 환경에서 인간과 자연스럽게 상호작용할 수 있습니다.

휴머노이드 로봇 - 인간의 신체 형태를 모방하여 설계된 로봇. 직립 이족 보행, 양팔 조작, 얼굴 표현 등을 통해 인간과 협력합니다.

휴머노이드 로봇의 분류

1) 외형별 분류

완전 휴머노이드: 머리, 몸통, 팔, 다리를 모두 갖춘 로봇 (예: Tesla Bot, Figure AI)

부분 휴머노이드: 상체만 인간형인 로봇 (예: PR2, Baxter)

준 휴머노이드: 인간과 유사한 행동만 모방하는 로봇

2) 용도별 분류

서비스 로봇: 호텔, 병원, 상점에서 서빙, 안내, 청소

제조 로봇: 조립, 용접, 검사 등 산업 작업

탐사 로봇: 재난 현장, 우주, 깊은 바다 탐사

돌봄 로봇: 노인 간호, 의료 보조, 재활 훈련

3) 동력별 분류

전기 모터: 가장 일반적, 정밀한 제어 가능

유압 시스템: 높은 토크, 무거운 물체 들기 가능

공압 시스템: 경량, 유연한 움직임

근육 와이어: 인간과 유사한 유연한 행동

휴머노이드 로봇의 핵심 기술 요소

1) 센싱 및 인식

• 시각: 카메라, 3D 비전

• 촉각: 터치 센서, 힘 센서

• 청각: 마이크, 음성 인식

2) 의사결정 (AI)

• 환경 이해 및 상황 판단

• 행동 계획 수립

• 자연어 상호작용

3) 동작 제어

• 관절 제어 알고리즘

• 안정성 유지 (자세 제어)

• 물체 조작 (손 제어)

4) 물리 설계

• 경량 소재

• 내구성 있는 관절

• 배터리 효율성

로봇명 개발사 주요 특징 용도

9. 컴퓨터 비전 (Computer Vision)

컴퓨터 비전의 정의

컴퓨터 비전은 이미지와 비디오로부터 의미 있는 정보를 자동으로 추출하고 이해하는 기술입니다. 인간의 시각 능력을 기계로 구현합니다.

컴퓨터 비전 - 디지털 이미지와 비디오를 처리하여 객체 인식, 장면 이해, 행동 분석 등을 수행하는 AI 기술. 심층 신경망을 활용합니다.

컴퓨터 비전의 주요 작업

1) 객체 탐지 (Object Detection)

이미지에서 특정 객체를 찾아내고 그 위치를 표시합니다.

주요 알고리즘

1
YOLO
(You Only Look Once)
2
SSD
(Single Shot MultiBox Detector)
3
R-CNN
(Region-based CNN)

YOLO (You Only Look Once)

이미지 전체를 한 번에 처리하여 객체 탐지 및 분류를 수행합니다. 실시간 처리에 유리합니다.

특징: 사진을 보자마자 "저기 강아지, 저기 자동차"라고 바로 말하는 것처럼, 빠르고 효율적입니다.

SSD (Single Shot MultiBox Detector)

다양한 크기의 필터를 사용하여 작은 객체부터 큰 객체까지 탐지합니다. YOLO와 R-CNN의 장점을 결합합니다.

R-CNN (Region-based CNN)

먼저 관심 영역(Region)을 제안한 후, 각 영역에 대해 분류합니다. 정확도가 높지만 처리 속도가 느립니다.

알고리즘 처리 방식 속도 정확도 활용
YOLO One-stage (전체 한번) 매우 빠름 중상 실시간 스트리밍
SSD One-stage (다중 스케일) 빠름 중상 모바일, 엣지
R-CNN Two-stage (제안후 분류) 느림 높음 정밀도 중시 분야

2) 의미론적 분할 (Semantic Segmentation)

이미지의 각 픽셀을 분류하여, 어느 부분이 어떤 객체인지 구분합니다.

예시: 도로 이미지를 "도로(검은색)", "보도(회색)", "차량(파란색)", "사람(빨간색)"으로 색칠하는 것.

3) 인스턴스 분할 (Instance Segmentation)

같은 종류의 객체라도 개별적으로 구분합니다. 예: 여러 사람을 각각 다르게 표시

4) 이미지 분류 (Image Classification)

전체 이미지를 하나의 카테고리로 분류합니다. 예: "고양이" / "개" / "새"

5) 얼굴 인식 (Face Recognition)

얼굴을 감지하고 신원을 확인합니다.

시각 데이터 플랫폼 (Visual Data Platform)

데이터 수집

• 카메라, 드론, 위성으로부터 대규모 이미지/비디오 수집

데이터 전처리

• 정규화, 증강(Augmentation), 불균형 처리

모델 개발

• CNN, Transformer 등 신경망 훈련

배포 및 모니터링

• 엣지 장치에 배포, 실시간 성능 모니터링

시험 포인트: 컴퓨터 비전의 핵심은 "특징 추출"입니다. 원시 픽셀에서 의미 있는 정보(객체, 위치, 속성)를 추출하는 과정입니다.

10. 딥페이크 (Deepfake)

딥페이크의 정의

딥페이크는 딥러닝 기술을 이용하여 실제 사람의 얼굴, 목소리, 행동을 합성하여 가짜 영상이나 음성을 생성하는 기술입니다. "Deep Learning" + "Fake"의 합성어입니다.

딥페이크 - 생성형 AI(GAN, VAE 등)를 사용하여 실제 같은 가짜 미디어(영상, 음성, 이미지)를 생성하는 기술. 매우 높은 현실성으로 인해 악용 위험이 큽니다.

딥페이크 생성 기술

1) GAN (Generative Adversarial Network)

생성자(Generator)와 판별자(Discriminator)가 경쟁하며 학습합니다. 생성자는 가짜를 만들고, 판별자는 진짜와 가짜를 구분합니다.

2) 얼굴 교체 (Face Swapping)

한 사람의 얼굴을 다른 사람의 영상에 합성합니다.

단계:

1) 소스 영상에서 얼굴 특징 추출

2) 타겟 영상의 얼굴 위치 감지

3) 소스 얼굴을 타겟에 맞게 변환

4) 경계선 블렌딩하여 자연스럽게 합성

3) 음성 합성 (Voice Synthesis)

특정인의 음성 데이터로부터 학습하여, 원하는 텍스트를 그 사람의 목소리로 변환합니다.

소스 데이터 → 특징 추출 → 생성 모델 → 합성 결과

딥페이크 탐지 방법

1) 생물학적 신호 분석 (Biological Signal Analysis)

자연스러운 신호 감지: 심박수, 눈 깜박임, 피부 혈류 변화

원리: 가짜 영상은 미세한 생물학적 신호를 재현하기 어렵습니다.

2) 주파수 분석 (Frequency Analysis)

이미지를 주파수 영역으로 변환하여 분석합니다. 합성 영상은 특정 주파수 패턴이 다릅니다.

3) 불일치 탐지 (Inconsistency Detection)

얼굴의 움직임, 머리 각도, 빛의 반사, 배경과의 조화도 등의 불일치를 찾습니다.

4) AI 기반 분류기 (AI-based Classifier)

실제 영상과 가짜 영상을 학습한 신경망으로 판별합니다.

5) 법정 과학적 분석 (Forensic Analysis)

이미지 메타데이터, 압축 아티팩트, 카메라 특성 분석

탐지 기법 원리 정확도 한계
생물학적 신호 심박수, 눈 깜박임 중상 압축된 영상에서 감지 어려움
주파수 분석 이상 패턴 감지 중상 최신 생성 모델에 취약
불일치 탐지 움직임, 조명 이상 중 숙련된 제작자에게 취약
AI 분류기 딥러닝 기반 높음 대적적 공격(adversarial attack)에 취약

딥페이크의 영향

부정적 영향

위험: 명예 훼손, 성적 학대, 정치적 거짓 정보 확산, 사기, 신뢰성 침해, 개인 사생활 침해

긍정적 영향

기회: 영화 제작, 특수 효과, 엔터테인먼트, 의료 훈련, 예술 표현, 장애인을 위한 보조 기술, 언어 학습

딥페이크 대응 방안

기술적 대응

• 탐지 기술 개발 및 고도화

• 블록체인 기반 인증 시스템

11. TensorFlow vs PyTorch

두 프레임워크의 개요

TensorFlow와 PyTorch는 딥러닝 모델을 개발하기 위한 오픈소스 라이브러리입니다. 둘 다 신경망 구축, 훈련, 배포를 위한 강력한 도구들을 제공합니다.

TensorFlow - Google이 개발한 오픈소스 머신러닝 플랫폼. 정적 그래프, 프로덕션 최적화에 강합니다.
PyTorch - Meta(Facebook)가 개발한 오픈소스 머신러닝 라이브러리. 동적 그래프, 연구 친화적입니다.

핵심 비교: 정적 vs 동적 그래프

TensorFlow: 정적 그래프 (Static Graph)

모델의 계산 그래프를 먼저 정의한 후, 데이터를 입력하여 실행합니다.

유사성: 조립 명령서를 먼저 작성한 후, 나중에 부품을 조립하는 것.

PyTorch: 동적 그래프 (Dynamic Graph)

데이터를 입력하면서 즉시 계산 그래프를 구성합니다. (Eager Execution)

유사성: 손으로 직접 부품을 만지면서 바로바로 조립하는 것.

전체 비교표

특성 TensorFlow PyTorch
개발사 Google Meta (Facebook)
그래프 방식 정적 (이전), 동적 (Eager Mode) 동적 (Eager Execution)
학습 곡선 가파름 (복잡한 문법) 완만함 (직관적)
코드 작성 다소 복잡 간결하고 명확
디버깅 어려움 쉬움 (일반 Python 디버거 사용)
연구 친화성 중간 높음
프로덕션 최적화 매우 강함 중간 (TorchScript 활용)
모바일 배포 TFLite 지원 제한적
커뮤니티 거대하고 성숙 빠르게 성장 중
자동 미분 (Autograd) GradientTape API 내장 Autograd
주요 프레임워크 Keras (고수준), TF-Hub Lightning, Hugging Face
성능 (GPU) 매우 최적화됨 우수 (점차 개선 중)

선택 기준

TensorFlow를 선택할 때

• 프로덕션 환경 배포 필요

• 대규모 병렬 처리 필요

• 모바일 또는 웹 배포

• Google 생태계와 통합

• 성숙한 커뮤니티 필요

PyTorch를 선택할 때

• 학습 및 실험 중심

• 복잡한 모델 구조 (RNN, 동적 길이 입력)

• 빠른 프로토타이핑 필요

• 연구 논문 구현

• 쉬운 디버깅 필요

코드 비교 예시

간단한 신경망 구현 (개념적 비교)

TensorFlow는 레이어를 선언적으로 정의하는 경향이 있고, PyTorch는 명령형으로 직접 제어합니다.

시험 포인트: TensorFlow는 "프로덕션 최적화", PyTorch는 "연구 친화성"에서 장점입니다. 현재는 둘 다 동적 모드를 지원하며 격차가 줄어들고 있습니다.

12. LangChain (간략 참조)

LangChain의 정의

LangChain은 대형 언어 모델(LLM)을 기반으로 한 애플리케이션을 구축하기 위한 Python/JavaScript 라이브러리입니다. 07장에서 상세히 다루었으므로, 여기서는 주요 개념만 참조합니다.

LangChain - LLM 애플리케이션 구축을 위한 모듈화된 프레임워크. Model I/O, Data Connection, Chains, Agents, Memory의 5가지 핵심 모듈로 구성.

5가지 핵심 모듈 (복습)

1
Model I/O
프롬프트 & 응답
2
Data Connection
외부 데이터
3
Chains
작업 연결
4
Agents
자율 의사결정
5
Memory
대화 기억

LangChain의 하위 프로젝트

1) LangSmith - 애플리케이션 모니터링 및 성능 분석

2) LangServe - LangChain 애플리케이션을 REST API로 배포

3) LangGraph - 상태 기계 기반 복잡한 워크플로우 관리

참조: LangChain은 에이전틱 AI 구현의 핵심 도구입니다. Model Context Protocol(MCP)과 함께 사용하면 외부 도구 접근이 용이합니다.

1) CNN (합성곱 신경망)

시계열 데이터 처리에 효과적. LSTM, GRU 등 개선된 버전 존재합니다.

4) Vision Transformer (ViT)

Transformer를 이미지 처리에 적용한 모델입니다.

5) Mixture of Experts (MoE)

여러 전문가 모듈을 조건부로 활용하여 효율성을 높입니다. GPT-4 등 최신 모델에 사용됩니다.
